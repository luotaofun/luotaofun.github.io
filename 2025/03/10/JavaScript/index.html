    概述

    起源于1995年，主要用于处理网页中的前端验证。

    • Netscape 与 Sun Microsystems（Java 的拥有者）合作后，将其正式命名为 JavaScript。
    • 这一命名主要是出于营销考虑，借用了当时非常流行的 Java 语言的热度，尽管两者语法和设计思想并不相同。

    2. 类型转换

    类型转换规则总结：

    1. 转换为字符串：
      • null → “null”
      • undefined → “undefined”
      • true → “true”
      • false → “false”
      • 数字 → 数字的字符串形式
      • 对象 → “[object Object]”
    2. 转换为数字：
      • 空字符串 → 0
      • 非空字符串 → 如果可以解析为数字则转换，否则NaN
      • true → 1
      • false → 0
      • null → 0
      • undefined → NaN
      • 对象 → 先转换为原始值，再转换为数字
    3. 转换为布尔值： 以下值转换为false：
      • 空字符串 “”
      • 数字 0
      • null
      • undefined
      • NaN 其他所有值都转换为true

    最佳实践

    1. 尽量使用显式转换（如String()、Number()、Boolean()）
    2. 避免依赖隐式转换
    3. 使用===而不是==，避免类型转换带来的问题
    4. 在进行数学运算前，确保操作数都是数字类型

    3. 自增运算符（++）

    区别总结： 1. ++a（前置自增）： - 先将a加1 - 再返回加1后的值 - 常用于：需要立即使用增加后的值时

    1. a++（后置自增）：
      • 先返回原始值
      • 再将a加1
      • 常用于：需要在使用当前值后再增加时

    最佳实践： 1. 单独使用时，++a 和 a++ 效果相同 2. 在表达式中使用时要特别注意前置和后置的区别

    4. 逻辑运算符（&&, ||, !）

    常见用途： 1. 短路求值

    1. 条件判断

    最佳实践： 1. 理解逻辑运算符返回的是值，不一定是布尔类型 2. 使用 && 进行条件判断时要注意短路特性 3. 使用 || 设置默认值是一种常见模式 4. 使用括号明确运算优先级

    1.逻辑运算符优先级和执行顺序

    5. 比较运算符（>, <, >=, <=, ==, ===）

    ==的类型转换规则总结：

    1. null和undefined：
      • null == undefined 返回 true
      • null和undefined与其他值比较都返回false
    2. 布尔值参与比较：
      • 布尔值先转换为数字（true→1，false→0）
      • 然后再和另一个操作数比较
    3. 数字和字符串比较：
      • 字符串转换为数字
      • 如果转换失败则变成NaN
    4. 对象和原始类型比较：
      • 对象先调用valueOf()或toString()转换为原始类型
      • 然后按照上述规则进行比较

    一、原型基础概念

    我们所创建的每一个函数，解析器都会向函数中添加一个属性prototype 这个属性对应着一个对象，这个对象就是我们所谓的原型对象 当函数以构造函数的形式调用时，它所创建的对象中都会有一个隐含的属性，它指向该构造函数的原型对象，我们可以通过__proto__来访问该属性 原型对象就相当于一个公共的区域，所有同一个类的实例都可以访问到这个原型对象，我们可以将对象中共有的内容，统一设置到原型对象中。 当我们访问对象的一个属性或方法时，它会先在对象自身中寻找，如果有则直接使用，如果没有则会去原型对象中寻找，如果找到则直接使用

    NodeList与HTMLCollection的区别

    • HTMLCollection：是动态集合，会随DOM的变化而自动更新
    • NodeList：通常是静态集合，不会自动更新（除了某些特定情况）
