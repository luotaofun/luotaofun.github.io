    对象（Object）- 存储相关信息📚

    一、对象的分类

    JavaScript中的对象可以分为三类：

    1. 内建对象（Built-in Objects）
    2. 宿主对象（Host Objects）
    3. 自定义对象（Custom Objects）

    1. 内建对象（Built-in Objects）

    这些对象是JavaScript语言标准的一部分，在任何JavaScript环境中都可用。

    // 1. 基本包装类型
let str = new String("Hello");      // 字符串对象
let num = new Number(123);          // 数字对象
let bool = new Boolean(true);       // 布尔对象

// 2. 引用类型
let obj = new Object();             // 对象
let arr = new Array(1, 2, 3);       // 数组
let date = new Date();              // 日期
let reg = new RegExp("\\d+");       // 正则表达式

// 3. 错误类型
let error = new Error("出错了");     // 错误对象
let syntaxError = new SyntaxError(); // 语法错误
let typeError = new TypeError();     // 类型错误

// 4. 全局对象
Math.PI;                            // 数学对象（不需要new）
JSON.stringify({name: "张三"});      // JSON对象（不需要new）

    2. 宿主对象（Host Objects）

    这些对象由JavaScript的运行环境提供，比如浏览器或Node.js。

    // 浏览器环境的宿主对象
window                              // 全局对象
document                            // 文档对象
console                             // 控制台对象
history                             // 历史对象
location                            // 地址栏对象
localStorage                        // 本地存储对象

// DOM对象
document.getElementById("myId")      // 元素对象
document.createElement("div")        // 创建DOM元素
document.querySelector(".myClass")   // 选择器

// BOM对象
window.innerHeight                   // 窗口高度
window.localStorage                  // 本地存储
window.navigator                     // 浏览器信息

// Node.js环境的宿主对象
process                             // 进程对象
Buffer                              // 二进制数据对象
require                             // 模块加载

    类型转换规则总结：

    1. 转换为字符串：

      • null → “null”
      • undefined → “undefined”
      • true → “true”
      • false → “false”
      • 数字 → 数字的字符串形式
      • 对象 → “[object Object]”

    2. 转换为数字：

      • 空字符串 → 0
      • 非空字符串 → 如果可以解析为数字则转换，否则NaN
      • true → 1
      • false → 0
      • null → 0
      • undefined → NaN
      • 对象 → 先转换为原始值，再转换为数字

    3. 转换为布尔值：
      以下值转换为false：

      • 空字符串 “”
      • 数字 0
      • null
      • undefined
      • NaN
        其他所有值都转换为true

    最佳实践

    1. 尽量使用显式转换（如String()、Number()、Boolean()）
    2. 避免依赖隐式转换
    3. 使用===而不是==，避免类型转换带来的问题
    4. 在进行数学运算前，确保操作数都是数字类型

    3. 自增运算符（++）

    // 1. 前置自增（++a）：先自增，再使用
let a = 1;
let b = ++a;    // a先加1，再赋值给b
console.log(a); // 2
console.log(b); // 2

// 2. 后置自增（a++）：先使用，再自增
let x = 1;
let y = x++;    // 先将x的值赋给y，再将x加1
console.log(x); // 2
console.log(y); // 1

// 在表达式中的区别
let num1 = 5;
let num2 = 5;
let d=10;
console.log(++num1 * 2); // 12 (先自增到6，再乘2)
console.log(num2++ * 2); // 10 (先乘2，再自增到6)
console.log(d++ + ++d +d) //34
// 让我们一步步分解这个表达式：
// 1. d++ + ++d + d
//    - d++ 是后置自增：先使用d的值(10)，然后d变成11
//    - ++d 是前置自增：先将d加1变成12，再使用12
//    - 最后的d 直接使用当前值12
// 所以计算过程是：10 + 12 + 12 = 34

let d = 10;
d = d++;
console.log(d);    // 结果：10
// 让我们分析这个过程：
// 1. d = d++
//    - d++ 是后置自增，会先返回原始值10
//    - 然后d会增加到11
//    - 但是这个11还没有被使用，d就被赋值了
//    - 赋值运算符=会将d++返回的原始值10赋给d
//    - 所以最终d的值仍然是10

    区别总结：

    1. ++a（前置自增）：

      • 先将a加1
      • 再返回加1后的值
      • 常用于：需要立即使用增加后的值时

    2. a++（后置自增）：

      • 先返回原始值
      • 再将a加1
      • 常用于：需要在使用当前值后再增加时

    最佳实践：

    1. 单独使用时，++a 和 a++ 效果相同
    2. 在表达式中使用时要特别注意前置和后置的区别

    let result = 1 || 2 && 3;    // 结果：1

// 1. && 优先级高于 ||
// 2. 从左到右执行

// 分解执行过程：
// 1 || 2 && 3
//    - 先计算 2 && 3（因为&&优先级更高）
//    - 2 && 3 的结果是 3（因为2是真值）
//    - 现在表达式变成：1 || 3

    5. 比较运算符（>, <, >=, <=, ==, ===）

// 特点：任何值和NaN比较都返回false，包括NaN自身

// 特点：undefined和数字比较时转换为NaN

null == 0;            // false
null >= 0;            // true
// 特点：null在比较时会转换为数字0，但==比较时有特殊规则

// 字符串比较
"11" > "2";           // false
"abc" > "abb";        // true
// 特点：按字符编码逐位比较
// 例如："11" > "2" 比较过程：
// - 比较第一位："1"的编码（49）< "2"的编码（50）
// - 所以整体结果为false，不再比较后续位置

// 不同类型比较
"23" > 22;            // true（字符串"23"会转换为数字23）
"abc" > 1;            // false（"abc"转换为数字是NaN）
true > 0;             // true（true转换为数字1）
false < 1;            // true（false转换为数字0）

    6. 相等运算符（==）和全等运算符（===）

    // 1. 全等运算符（===）
// - 比较值和类型都要相同
5 === 5;              // true
5 === "5";           // false（类型不同）
null === undefined;   // false（类型不同）
NaN === NaN;         // false（NaN不等于任何值，包括自身,可以通过isNaN()判断是否是NaN）

// 2. 相等运算符（==）
// - 会进行类型转换后比较
5 == "5";            // true（字符串"5"会转换为数字5）
true == 1;           // true（true转换为数字1）
false == 0;          // true（false转换为数字0）
null == undefined;   // true（特殊规则：undefined是null衍生过来的）

// 3.2 布尔值和其他类型
true == 1;           // true（布尔值转换为数字）
false == "";         // true（都转换为数字0）
false == [];         // true（[]转换为数字0）

// 3.3 数字和字符串
1 == "1";            // true（字符串转换为数字）
1 == "one";          // false（"one"转换为NaN）

// 3.4 对象和原始类型
[1] == 1;            // true（[1]转换为"1"，然后转换为数字1）
["1"] == 1;          // true（同上）
[1,2] == "1,2";      // true（数组转换为字符串后比较）

    ==的类型转换规则总结：

    1. null和undefined：

      • null == undefined 返回 true
      • null和undefined与其他值比较都返回false

    2. 布尔值参与比较：

      • 布尔值先转换为数字（true→1，false→0）
      • 然后再和另一个操作数比较

    3. 数字和字符串比较：

      • 字符串转换为数字
      • 如果转换失败则变成NaN

    4. 对象和原始类型比较：

      • 对象先调用valueOf()或toString()转换为原始类型
      • 然后按照上述规则进行比较

    1. 引用类型的拷贝

    // 1. 浅拷贝

let shallowCopy1 = Object.assign({}, original);

// 方法二：展开运算符
let shallowCopy2 = { ...original };

// 2. 深拷贝
// 方法一：JSON转换（有局限性）

// 方法二：递归实现
function deepClone(obj) {
    if (obj === null || typeof obj !== 'object') return obj;
    let copy = Array.isArray(obj) ? [] : {};
    for (let key in obj) {
        if (obj.hasOwnProperty(key)) {
            copy[key] = deepClone(obj[key]);
        }
    }
    return copy;
}

    六、apply 和 call 方法

    用途

    apply 和 call 都是函数对象的方法，用于改变函数执行时的上下文（即改变函数内部的 this 指向）。

    语法

    func.call(thisArg, param1, param2, ...)
func.apply(thisArg, [param1, param2, ...])

    区别

    • call 方法：参数需要一个一个传递
    • apply 方法：参数以数组形式传递

    示例

    1. 基本使用

    const person = {
    name: '张三',
    sayHi: function(greeting) {
        console.log(`${greeting}，我是${this.name}`);
    }
};

const other = {
    name: '李四'
};

// 使用 call
person.sayHi.call(other, '你好');  // 输出：你好，我是李四

// 使用 apply
person.sayHi.apply(other, ['你好']); // 输出：你好，我是李四

    2. 实际应用场景

    2.1 借用数组方法
    // 将类数组转换为真正的数组
function convert() {
    return Array.prototype.slice.call(arguments);
}
console.log(convert(1, 2, 3)); // 输出：[1, 2, 3]

// 或使用 apply 找出数组最大值
const numbers = [5, 6, 2, 3, 7];
const max = Math.max.apply(null, numbers);
console.log(max); // 输出：7
    2.2 继承
    function Animal(name) {
    this.name = name;
}

function Cat(name, color) {
    // 调用 Animal 构造函数，并将 this 指向 Cat 实例
    Animal.call(this, name);
    this.color = color;
}

const cat = new Cat('咪咪', '白色');
console.log(cat.name, cat.color); // 输出：咪咪 白色

    关键点

    1. this 绑定

      • 第一个参数决定了函数内部的 this 指向
      • 如果第一个参数为 null 或 undefined，this 会指向全局对象（非严格模式）或保持为 null/undefined（严格模式）

    2. 参数传递

      • call：func.call(thisArg, arg1, arg2, ...)
      • apply：func.call(thisArg, [arg1, arg2, ...])

    3. 使用场景

      • 借用其他对象的方法
      • 继承时调用父类构造函数
      • 调用需要特定 this 值的函数

    最佳实践

    1. 当参数数量固定时，优先使用 call，代码更直观
    2. 当参数数量不固定或参数本身就是数组时，使用 apply
    3. 在现代 JavaScript 中，可以考虑使用展开运算符 ... 代替 apply：
      const numbers = [5, 6, 2, 3, 7];
const max = Math.max(...numbers); // 替代 Math.max.apply(null, numbers)

    注意事项

    1. 在严格模式下，如果第一个参数为 null 或 undefined，this 不会被转换为全局对象
    2. 这两个方法都会立即执行函数
    3. 如果需要稍后执行，可以考虑使用 bind 方法

    1. for…of

    用途

    for...of 循环用于遍历可迭代对象（如数组、字符串、Map、Set等）。

    使用方法

    for (const element of iterable) {
    // 处理每个元素
}

    参数说明

    • iterable: 任何可迭代对象。

    返回值

    • 无返回值。

    示例

    const array = [1, 2, 3];
for (const num of array) {
    console.log(num); // 输出 1, 2, 3
}

    2. forEach

    用途

    forEach 方法用于对数组的每个元素执行一次提供的函数。

    使用方法

    array.forEach((element, index, array) => {
    // 处理每个元素
});

    参数说明

    • element: 当前处理的数组元素。
    • index: 当前元素的索引（可选）。
    • array: 调用 forEach 的数组（可选）。

    返回值

    • 无返回值。

    示例

    const array = [1, 2, 3];
array.forEach((num) => {
    console.log(num); // 输出 1, 2, 3
});

    3. map

    用途

    map 方法创建一个新数组，结果是对原数组中的每个元素调用一个提供的函数后的返回值。

    使用方法

    const newArray = array.map((element, index, array) => {
    // 返回处理后的元素
});

    参数说明

    • element: 当前处理的数组元素。
    • index: 当前元素的索引（可选）。
    • array: 调用 map 的数组（可选）。

    返回值

    • 返回一个新数组。

    示例

    const array = [1, 2, 3];
const doubled = array.map(num => num * 2);
console.log(doubled); // 输出 [2, 4, 6]

    4. filter

    用途

    filter 方法创建一个新数组，包含所有通过测试的元素。

    使用方法

    const newArray = array.filter((element, index, array) => {
    // 返回 true 或 false
});

    参数说明

    • element: 当前处理的数组元素。
    • index: 当前元素的索引（可选）。
    • array: 调用 filter 的数组（可选）。

    返回值

    • 返回一个新数组，包含所有通过测试的元素。

    示例

    const array = [1, 2, 3, 4];
const evens = array.filter(num => num % 2 === 0);
console.log(evens); // 输出 [2, 4]

    5. reduce

    用途

    reduce 方法对数组中的每个元素执行一个 reducer 函数，最终计算出一个单一的值。

    使用方法

    const result = array.reduce((accumulator, currentValue, index, array) => {
    // 返回更新后的累加器
}, initialValue);

    参数说明

    • accumulator: 累加器，累计回调的返回值。
    • currentValue: 当前处理的数组元素。
    • index: 当前元素的索引（可选）。
    • array: 调用 reduce 的数组（可选）。
    • initialValue: 作为第一次调用回调时第一个参数的值（可选）。

    返回值

    • 返回最终的累加值。

    示例

    const array = [1, 2, 3, 4];
const sum = array.reduce((acc, num) => acc + num, 0);
console.log(sum); // 输出 10

    6. 数组去重实现

    • 对于简单数组去重，优先使用 Set 方法
    • 需要保持数组原有顺序时，可以使用 filter 方法
    • 需要在遍历过程中进行复杂操作时，可以使用 splice 方法

    方法一：使用双重循环和splice

    这种方法通过比较每个元素与其后的所有元素，发现重复就删除。

    let array = [1, 2, 2, 3, 4, 5];
for (let i = 0; i < array.length; i++) {
    // 遍历该元素后的所有元素依次比较
    for (let j = i + 1; j < array.length; j++) {
        if (array[i] === array[j]) {
            array.splice(j, 1); // 删除重复元素
            // 因为数组长度减少了1，所以索引也要减1。即删除了当前元素，后一个元素顶上来了,此时比较下一个元素将会漏一个元素没比较
            j--; 

        }
    }
}
console.log(array); // 输出 [1, 2, 3, 4, 5]

    方法二：使用Set（推荐）

    let array = [1, 2, 2, 3, 4, 5];
array = [...new Set(array)];
console.log(array); // 输出 [1, 2, 3, 4, 5]

    方法三：使用filter和indexOf

    let array = [1, 2, 2, 3, 4, 5];
// 元素在数组中首次出现的位置等于其当前索引
array = array.filter((item, index) => array.indexOf(item) === index);
console.log(array); // 输出 [1, 2, 3, 4, 5]
