    Setup语法糖

    基本使用方法

    1. 普通的Composition API写法
    <template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">增加</button>
  </div>
</template>

<script>
import { ref, onMounted } from 'vue'

export default {
  name:"counter",
  setup() {
    const count = ref(0)
    
    function increment() {
      count.value++
    }
    
    onMounted(() => {
      console.log('组件已挂载')
    })
    
    // 必须返回要在模板中使用的内容
    return {
      count,
      increment
    }
  }
}
</script>
    2. 使用Setup语法糖的写法

    Setup语法糖的特性

    1. 自动暴露变量和函数

    在 <script setup> 中定义的顶层变量、函数和导入的内容都会自动暴露给模板，不需要通过 return 语句返回。

    <script setup>
// 导入的组件会自动注册
import HelloWorld from './HelloWorld.vue'
import { ref } from 'vue'

// 声明的变量和函数自动可用于模板
const message = ref('Hello')
function updateMessage() {
  message.value = 'Updated'
}
</script>

<template>
  <!-- 可以直接使用上面定义的变量和方法 -->
  <div>{{ message }}</div>
  <button @click="updateMessage">更新</button>
  <!-- 导入的组件可直接使用 -->
  <HelloWorld />
</template>
    2. 使用 defineProps 和 defineEmits

    在setup语法糖中，使用 defineProps 和 defineEmits 编译宏来声明props和emits，它们会自动导入并在编译时进行处理。

    <script setup>
// 声明props
const props = defineProps({
  title: String,
  likes: {
    type: Number,
    default: 0
  }
})

// 声明emits
const emit = defineEmits(['update', 'delete'])

// 使用emit
function handleClick() {
  emit('update', { id: 1, value: 'new value' })
}
</script>

<template>
  <h1>{{ title }}</h1>
  <p>Likes: {{ likes }}</p>
  <button @click="handleClick">更新</button>
</template>
    3. 使用 defineExpose

    默认情况下，<script setup> 组件是”封闭”的，即通过模板ref或$parent获取的组件实例不会暴露在 <script setup> 中声明的任何属性。要显式暴露属性，使用 defineExpose 编译宏：

    <script setup>
import { ref } from 'vue'

const count = ref(0)
const increment = () => count.value++

// 显式暴露属性和方法给父组件
defineExpose({
  count,
  increment
})
</script>

    这就像是在”保密柜”中，你可以选择性地公开某些文件供他人查看。

    4. 使用 useSlots 和 useAttrs

    访问slots和attrs可以使用 useSlots 和 useAttrs 辅助函数：

    <script setup>
import { useSlots, useAttrs } from 'vue'

const slots = useSlots()
const attrs = useAttrs()

console.log(slots)
console.log(attrs)
</script>
    5. 顶层 await 支持

    <script setup> 支持在顶层使用 await，这会自动将组件变为异步组件：

    <script setup>
// 顶层await - 组件会自动变为异步组件
const data = await fetch('/api/data').then(r => r.json())
</script>

    这就像是”边走边吃”——你可以在准备组件的过程中等待异步数据加载完成。

    与普通setup函数的区别

    特性 普通setup函数 Setup语法糖
    代码量 较多，需要export default 更少，更简洁
    暴露变量 需要手动return 自动暴露顶层变量
    TypeScript支持 需要额外配置 更好的类型推断
    组件注册 需要在components选项中注册 导入即自动注册
    生命周期钩子 需要使用指定API 直接导入使用
    获取实例 可以通过this访问 没有this，更函数式

    实际开发示例

    示例1：计数器组件
    <template>
  <div class="counter">
    <h2>{{ title }}</h2>
    <p>当前计数: {{ count }}</p>
    <div class="buttons">
      <button @click="increment">+1</button>
      <button @click="decrement">-1</button>
      <button @click="reset">重置</button>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, watch } from 'vue'

// 接收props
const props = defineProps({
  title: {
    type: String,
    default: '计数器'
  },
  initialValue: {
    type: Number,
    default: 0
  },
  step: {
    type: Number,
    default: 1
  }
})

// 声明事件
const emit = defineEmits(['change', 'reset'])

// 响应式状态
const count = ref(props.initialValue)

// 计算属性
const doubleCount = computed(() => count.value * 2)

// 方法
function increment() {
  count.value += props.step
  emit('change', count.value)
}

function decrement() {
  count.value -= props.step
  emit('change', count.value)
}

function reset() {
  count.value = props.initialValue
  emit('reset')
}

// 监听变化
watch(count, (newValue) => {
  console.log(`计数已更改为: ${newValue}`)
})

// 暴露给父组件的方法和属性
defineExpose({
  count,
  reset,
  doubleCount
})
</script>

  <div class="user-profile">
    <div v-if="loading">加载中...</div>
    <div v-else-if="error">加载失败: {{ error }}</div>

<script setup>
import { ref, onMounted } from 'vue'

const props = defineProps({
  userId: {
    type: Number,
    required: true
  }
})

const user = ref(null)
const loading = ref(true)
const error = ref(null)

// 使用async/await获取数据
async function fetchUserData() {
  loading.value = true
  error.value = null
  
  try {
    const response = await fetch(`https://api.example.com/users/${props.userId}`)
    if (!response.ok) {
      throw new Error(`API请求失败: ${response.status}`)
    }
    user.value = await response.json()
  } catch (err) {
    error.value = err.message
    console.error('获取用户数据失败:', err)
  } finally {
    loading.value = false
  }
}

// 当组件挂载时获取数据
onMounted(fetchUserData)

// 监听userId的变化重新获取数据
watch(() => props.userId, fetchUserData)
</script>

    常见问题解答

    问题1：<script setup> 中如何访问组件实例（this）？

    答：在<script setup>中没有this。这是函数式编程的一部分，所有需要的内容都应通过导入的API和组合函数获取。

    问题2：如何使用生命周期钩子？

    答：直接导入并调用生命周期钩子函数：

    <script setup>
import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  console.log('组件已挂载')
})

onUnmounted(() => {
  console.log('组件已卸载')
})
</script>

    问题3：如何使用依赖注入（provide/inject）？

    答：

    <!-- 提供方 -->
<script setup>
import { provide, ref } from 'vue'

const theme = ref('light')
provide('theme', theme) // 提供响应式值
</script>

<!-- 注入方 -->
<script setup>
import { inject } from 'vue'

const theme = inject('theme', 'light') // 第二个参数是默认值
</script>

    响应式系统

    Vue 3的响应式系统是框架的核心特性，它允许数据变化自动触发视图更新。在Composition API中，响应式系统通过ref、reactive等API实现。

    基本响应式API

    1. ref：创建一个响应式的数据引用，适用于基本类型和对象

      import { ref } from 'vue'

const count = ref(0)  // 基本类型
console.log(count.value)  // 必须使用.value访问

const user = ref({ name: 'Alice' })  // 对象
console.log(user.value.name)

    2. reactive：创建一个响应式对象，仅适用于对象（不能用于原始值）

      import { reactive } from 'vue'

const state = reactive({
  count: 0,
  user: { name: 'Bob' }
})
console.log(state.count)  // 直接访问，不需要.value

    生命周期钩子

    Vue 3中的生命周期钩子允许你在组件不同阶段执行代码。在Composition API中，生命周期钩子是以on开头的函数。

    import { onMounted, onUpdated, onUnmounted } from 'vue'

// 在setup中使用生命周期钩子
onMounted(() => {
  console.log('组件已挂载')
})

onUpdated(() => {
  console.log('组件已更新')
})

onUnmounted(() => {
  console.log('组件已卸载')
})

    ref和reactive的区别

    • ref定义基本数据类型、对象类型数据。reactive定义对象类型数据
    • ref创建的变量必须.value。reactive重新分配一个新对象会失去响应式，可以用Object.assign去整体替换
    • 定义层级比较深的响应式对象推荐用reactive
    • ref底层也是依赖reactive
    1. reactive
      • 用途：用于创建一个响应式的对象。
      • 特性：只能用于对象（包括数组），不能用于基本类型（如字符串、数字、布尔值等）。
      • 响应式原理：通过 Proxy 对象实现深度响应式。
    2. ref
      • 用途：用于创建一个响应式的引用对象，可以包含任何类型的值，包括基本类型。
      • 特性：对于对象，ref 会自动将其内部值转换为 reactive 对象。
      • 响应式原理：通过 Proxy 对象实现响应式，对于基本类型，通过 get 和 set 拦截器实现响应式。

    重新分配新对象时的行为

    1. reactive 对象
      • 直接重新赋值：如果直接将 reactive 对象重新赋值为一个新的对象，新的对象不会自动成为响应式对象。
      • 解决方案：需要使用 reactive 包装新的对象，或者直接修改现有对象的属性。
    2. ref 对象
      • 直接重新赋值：如果将 ref 对象的 .value 属性重新赋值为一个新的对象，新的对象会自动成为响应式对象。
      • 解决方案：可以直接重新赋值 .value 属性，或者使用 reactive 包装新的对象。

    Vue3核心概念

    响应式系统

    技术解释：Vue3的响应式系统基于ES6的Proxy，能够直接监听对象和数组的变化。

    生活案例：想象一个智能家居系统。当你调整温度设定（数据变化）时，空调会自动响应并调整工作模式（UI更新）。Vue3的响应式系统就像这个智能家居的控制中心，监测所有设定的变化并通知相关设备做出反应。

    组件化

    技术解释：Vue应用由独立、可复用的组件构建而成，每个组件封装自己的HTML、CSS和JavaScript。

    生活案例：组件就像乐高积木。每个积木（组件）有特定的形状和功能，可以独立存在，也可以与其他积木组合创建复杂结构。一个页面就是由这些不同功能的”积木”组合而成的。

    组件通信方式

    组件通信方式之v-model

    技术解释：Vue3中的v-model是props和emit的语法糖，用于在父子组件间实现双向绑定。

    生活案例：想象父母（父组件）给孩子（子组件）零花钱。v-model就像一个特殊的钱包：父母放钱进去（props传值），孩子花钱后会自动更新钱包余额并通知父母（emit事件）。这样父母始终知道钱包里有多少钱，实现了”双向通信”。

    代码示例：

}

    组合式API

    shallowRef与shallowReactive

    技术解释：shallowRef和shallowReactive创建浅层响应式对象，只有对象的顶层属性是响应式的，嵌套对象不会被转换。

    生活案例：想象一个邮箱系统。普通的ref/reactive就像完整的邮件追踪服务，从寄件到收件的每一步都会通知你（深层响应）。而shallow版本就像只追踪包裹是否已送达（顶层响应），但不会告诉你包裹内部物品的状态变化。

    代码示例：

    <template>
  <div>
    <h2>邮件系统</h2>
    <button @click="updateDeep">更新深层属性 (reactive)</button>
    <button @click="updateShallow">更新深层属性 (shallowReactive)</button>
    <pre>{{ normalPackage }}</pre>
    <pre>{{ shallowPackage }}</pre>
  </div>
</template>

<script setup>
import { reactive, shallowReactive } from 'vue'

// 完全响应式
const normalPackage = reactive({
  status: '运送中',
  details: {
    items: ['书', '衣服'],
    weight: '2kg'
  }
})

// 浅层响应式
const shallowPackage = shallowReactive({
  status: '运送中',
  details: {
    items: ['电子产品', '文具'],
    weight: '1kg'
  }
})

function updateDeep() {
  // 这会触发视图更新
  normalPackage.details.items.push('新物品')
}

function updateShallow() {
  // 这不会触发视图更新，因为details是非响应式的
  shallowPackage.details.items.push('新物品')
  
  // 但这会触发更新，因为status是顶层属性
  // shallowPackage.status = '已送达'
}
</script>

    readonly与shallowReadonly

    技术解释：readonly创建一个深层只读代理，任何嵌套属性都不可修改，而shallowReadonly只使顶层属性为只读。

    生活案例：想象一份合同文档。readonly就像一份完全盖章的合同，所有条款和附件都不能修改（深层只读）。而shallowReadonly就像只有主要条款不能修改，但附件内容可以调整（浅层只读）。

    代码示例：

    <template>
  <div>
    <h2>合同系统</h2>
    <button @click="tryModify">尝试修改合同</button>
    <pre>{{ contractInfo }}</pre>
  </div>
</template>

<script setup>
import { reactive, readonly, shallowReadonly } from 'vue'

// 原始合同数据
const originalContract = reactive({
  title: '服务协议',
  signDate: '2023-05-15',
  terms: {
    duration: '12个月',
    payment: '5000元/月'
  }
})

// 完全只读版本
const readonlyContract = readonly(originalContract)

// 浅层只读版本
const shallowReadonlyContract = shallowReadonly(originalContract)

// 用于展示的合同信息
const contractInfo = reactive({
  original: originalContract,
  readonly: readonlyContract,
  shallowReadonly: shallowReadonlyContract
})

function tryModify() {
  try {
    // 这会成功
    originalContract.title = '更新的服务协议'
    
    try {
      // 这会失败并显示警告
      readonlyContract.title = '尝试修改只读合同'
    } catch (e) {
      console.log('无法修改readonly合同的顶层属性')
    }
    
    try {
      // 这会失败并显示警告
      readonlyContract.terms.duration = '24个月'
    } catch (e) {
      console.log('无法修改readonly合同的嵌套属性')
    }
    
    try {
      // 这会失败并显示警告
      shallowReadonlyContract.title = '尝试修改浅层只读合同'
    } catch (e) {
      console.log('无法修改shallowReadonly合同的顶层属性')
    }
    
    // 这会成功，因为嵌套属性不是只读的
    shallowReadonlyContract.terms.duration = '24个月'
    
    console.log('修改操作完成')
  } catch (e) {
    console.error(e)
  }
}
</script>

    toRaw与markRaw

    技术解释：toRaw返回响应式对象的原始对象，markRaw标记一个对象永远不会被转为响应式。

    生活案例：想象一个智能家居系统。toRaw就像临时关闭智能功能，获取设备的原始状态进行维护；markRaw则像永久将某些设备（如老式电视）排除在智能系统外，它们的状态变化永远不会触发自动化操作。

    代码示例：

    <template>
  <div>
    <h2>智能家居控制面板</h2>
    <button @click="updateDevice">更新设备状态</button>
    <button @click="getOriginalState">获取原始状态</button>
    <button @click="addNewDevice">添加新设备</button>
    <pre>{{ deviceInfo }}</pre>
  </div>
</template>

<script setup>
import { reactive, toRaw, markRaw, ref } from 'vue'

// 响应式智能设备列表
const smartDevices = reactive({
  livingRoom: {
    lights: { status: 'on', brightness: 80 },
    thermostat: { status: 'on', temperature: 22 }
  },
  kitchen: {
    lights: { status: 'off', brightness: 0 },
    refrigerator: { status: 'on', temperature: 4 }
  }
})

// 用于展示的信息
const deviceInfo = ref({
  smartDevices,
  rawState: null,
  updateCount: 0
})

// 更新设备状态
function updateDevice() {
  smartDevices.livingRoom.lights.brightness += 5
  if (smartDevices.livingRoom.lights.brightness > 100) {
    smartDevices.livingRoom.lights.brightness = 100
  }
  deviceInfo.value.updateCount++
}

// 获取原始状态（不触发响应式更新）
function getOriginalState() {
  // 使用toRaw获取原始对象
  const rawDevices = toRaw(smartDevices)
  deviceInfo.value.rawState = rawDevices
  
  // 修改原始对象不会触发视图更新
  rawDevices.livingRoom.thermostat.temperature = 25
}

// 添加新设备（使用markRaw标记为非响应式）
function addNewDevice() {
  // 创建一个永远不会成为响应式的设备对象
  const oldTV = markRaw({
    status: 'on',
    channel: 5,
    volume: 30
  })
  
  // 即使添加到响应式对象中，oldTV也不会变成响应式的
  smartDevices.livingRoom.tv = oldTV
  
  // 这个修改不会触发更新
  setTimeout(() => {
    oldTV.channel = 8
    deviceInfo.value.updateCount++
  }, 1000)
}
</script>

    customRef的使用

    技术解释：customRef允许创建自定义的响应式引用，可以显式控制依赖追踪和触发更新。

    生活案例：想象一个智能搜索框。普通的ref就像即时搜索，每输入一个字符就立即搜索；而customRef就像带有防抖功能的搜索框，等你停止输入一段时间后才执行搜索，避免频繁无效的搜索请求。

    代码示例：

    <template>
  <div>
    <h2>智能搜索</h2>
    <input 
      type="text" 
      placeholder="输入搜索内容..." 
      v-model="searchQuery"
    />
    <p>当前搜索: {{ searchQuery }}</p>
    <p>搜索结果将在您停止输入后显示</p>
    <div v-if="isSearching">搜索中...</div>
    <ul v-else-if="searchResults.length">
      <li v-for="(result, index) in searchResults" :key="index">
        {{ result }}
      </li>
    </ul>
    <p v-else-if="searchQuery">无搜索结果</p>
  </div>
</template>

<script setup>
import { customRef, ref } from 'vue'

// 创建一个带防抖功能的ref
function useDebouncedRef(value, delay = 500) {
  let timeout
  return customRef((track, trigger) => {
    return {
      get() {
        // 追踪依赖
        track()
        return value
      },
      set(newValue) {
        // 清除之前的定时器
        clearTimeout(timeout)
        // 设置新的定时器，延迟触发更新
        timeout = setTimeout(() => {
          value = newValue
          // 触发更新
          trigger()
        }, delay)
      }
    }
  })
}

// 使用自定义的防抖ref
const searchQuery = useDebouncedRef('')
const isSearching = ref(false)
const searchResults = ref([])

// 监听searchQuery的变化
watch(searchQuery, async (newQuery) => {
  if (newQuery.trim()) {
    isSearching.value = true
    // 模拟API搜索
    try {
      await new Promise(resolve => setTimeout(resolve, 1000))
      // 模拟搜索结果
      searchResults.value = [
        `关于 "${newQuery}" 的结果 1`,
        `关于 "${newQuery}" 的结果 2`,
        `关于 "${newQuery}" 的结果 3`
      ]
    } catch (error) {
      console.error('搜索出错:', error)
      searchResults.value = []
    } finally {
      isSearching.value = false
    }
  } else {
    searchResults.value = []
  }
})
</script>

    高频面试题解析

    1. Vue3中的响应式原理是什么？

    技术解释：Vue3的响应式原理基于ES6的Proxy，能够直接监听对象和数组的变化。当数据发生变化时，Vue会自动更新相关的DOM，保持UI和数据的同步。

    生活案例：想象一个智能家居系统。当你调整温度设定（数据变化）时，空调会自动响应并调整工作模式（UI更新）。Vue3的响应式系统就像这个智能家居的控制中心，监测所有设定的变化并通知相关设备做出反应。

    2. Vue3中的组件通信方式有哪些？

    技术解释：Vue3中的组件通信方式包括props和emit、v-model、$refs和$parent、provide/inject、$attrs和插槽系统。

    生活案例：想象一个家族WhatsApp群组。家族长辈（祖先组件）在群里发布重要信息（provide数据），所有家族成员（后代组件）无论层级关系如何，都能直接接收到这些信息（inject数据），而不需要通过中间人传话。

    3. Vue3中的组合式API是什么？

    技术解释：Vue3中的组合式API是一组基于函数的API，允许在不使用选项式API的情况下组织和重用组件逻辑。它包括ref、reactive、computed、watch、生命周期钩子等函数。

    生活案例：想象一个智能家居系统。组合式API就像一个工具箱，里面有各种工具（函数），你可以根据需要选择和组合使用，构建出不同的智能功能。

    4. Vue3中的Teleport是什么？

    技术解释：Teleport是一个内置组件，允许将组件的一部分DOM”传送”到组件DOM层次结构之外的DOM节点中。

    生活案例：想象一个购物网站的客服聊天窗口。虽然聊天功能逻辑属于产品页面组件，但聊天窗口需要固定在屏幕右下角，不受页面滚动影响。Teleport就像一个传送门，让聊天窗口的UI可以”传送”到页面底部，而逻辑仍留在原组件中。

    5. Vue3中的Suspense是什么？

    技术解释：Suspense是一个内置组件，用于协调对异步依赖的处理，可以在等待异步组件时显示加载状态。

    生活案例：想象一个在线电影平台。当用户点击观看电影时，需要等待视频加载完成。Suspense就像一个耐心的影院经理，在视频加载期间显示”加载中”的画面，等视频准备好后再开始播放。
