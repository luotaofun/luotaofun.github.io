Setup语法糖

基本使用方法

1. 普通的Composition API写法
<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">增加</button>
  </div>
</template>

<script>
import { ref, onMounted } from 'vue'

export default {
  name:"counter",
  setup() {
    const count = ref(0)
    
    function increment() {
      count.value++
    }
    
    onMounted(() => {
      console.log('组件已挂载')
    })
    
    // 必须返回要在模板中使用的内容
    return {
      count,
      increment
    }
  }
}
</script>
2. 使用Setup语法糖的写法
<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">增加</button>
  </div>
</template>
<script lang="ts" >
   export default{
        name:"counter"
   }
</script>
<script setup>
import { ref, onMounted } from 'vue'

// 不再需要export default和setup函数
const count = ref(0)

function increment() {
  count.value++
}

onMounted(() => {
  console.log('组件已挂载')
})

// 不需要显式返回，顶层变量和函数自动暴露给模板
</script>

Setup语法糖的特性

1. 自动暴露变量和函数

在 <script setup> 中定义的顶层变量、函数和导入的内容都会自动暴露给模板，不需要通过 return 语句返回。

<script setup>
// 导入的组件会自动注册
import HelloWorld from './HelloWorld.vue'
import { ref } from 'vue'

// 声明的变量和函数自动可用于模板
const message = ref('Hello')
function updateMessage() {
  message.value = 'Updated'
}
</script>

<template>
  <!-- 可以直接使用上面定义的变量和方法 -->
  <div>{{ message }}</div>
  <button @click="updateMessage">更新</button>
  <!-- 导入的组件可直接使用 -->
  <HelloWorld />
</template>
2. 使用 defineProps 和 defineEmits

在setup语法糖中，使用 defineProps 和 defineEmits 编译宏来声明props和emits，它们会自动导入并在编译时进行处理。

<script setup>
// 声明props
const props = defineProps({
  title: String,
  likes: {
    type: Number,
    default: 0
  }
})

// 声明emits
const emit = defineEmits(['update', 'delete'])

// 使用emit
function handleClick() {
  emit('update', { id: 1, value: 'new value' })
}
</script>

<template>
  <h1>{{ title }}</h1>
  <p>Likes: {{ likes }}</p>
  <button @click="handleClick">更新</button>
</template>
3. 使用 defineExpose

默认情况下，<script setup> 组件是”封闭”的，即通过模板ref或$parent获取的组件实例不会暴露在 <script setup> 中声明的任何属性。要显式暴露属性，使用 defineExpose 编译宏：

<script setup>
import { ref } from 'vue'

const count = ref(0)
const increment = () => count.value++

// 显式暴露属性和方法给父组件
defineExpose({
  count,
  increment
})
</script>

这就像是在”保密柜”中，你可以选择性地公开某些文件供他人查看。

4. 使用 useSlots 和 useAttrs

访问slots和attrs可以使用 useSlots 和 useAttrs 辅助函数：

<script setup>
import { useSlots, useAttrs } from 'vue'

const slots = useSlots()
const attrs = useAttrs()

console.log(slots)
console.log(attrs)
</script>
5. 顶层 await 支持

<script setup> 支持在顶层使用 await，这会自动将组件变为异步组件：

<script setup>
// 顶层await - 组件会自动变为异步组件
const data = await fetch('/api/data').then(r => r.json())
</script>

这就像是”边走边吃”——你可以在准备组件的过程中等待异步数据加载完成。

与普通setup函数的区别

特性 普通setup函数 Setup语法糖
代码量 较多，需要export default 更少，更简洁
暴露变量 需要手动return 自动暴露顶层变量
TypeScript支持 需要额外配置 更好的类型推断
组件注册 需要在components选项中注册 导入即自动注册
生命周期钩子 需要使用指定API 直接导入使用
获取实例 可以通过this访问 没有this，更函数式

实际开发示例

示例1：计数器组件
<template>
  <div class="counter">
    <h2>{{ title }}</h2>
    <p>当前计数: {{ count }}</p>
    <div class="buttons">
      <button @click="increment">+1</button>
      <button @click="decrement">-1</button>
      <button @click="reset">重置</button>
    </div>
  </div>
</template>

<script setup>
import { ref, computed, watch } from 'vue'

// 接收props
const props = defineProps({
  title: {
    type: String,
    default: '计数器'
  },
  initialValue: {
    type: Number,
    default: 0
  },
  step: {
    type: Number,
    default: 1
  }
})

// 声明事件
const emit = defineEmits(['change', 'reset'])

// 响应式状态
const count = ref(props.initialValue)

// 计算属性
const doubleCount = computed(() => count.value * 2)

// 方法
function increment() {
  count.value += props.step
  emit('change', count.value)
}

function decrement() {
  count.value -= props.step
  emit('change', count.value)
}

function reset() {
  count.value = props.initialValue
  emit('reset')
}

// 监听变化
watch(count, (newValue) => {
  console.log(`计数已更改为: ${newValue}`)
})

// 暴露给父组件的方法和属性
defineExpose({
  count,
  reset,
  doubleCount
})
</script>

示例2：API数据获取组件
<template>
  <div class="user-profile">
    <div v-if="loading">加载中...</div>
    <div v-else-if="error">加载失败: {{ error }}</div>
    <div v-else class="user-data">
      <img :src="user.avatar" alt="用户头像">
      <h2>{{ user.name }}</h2>
      <p>Email: {{ user.email }}</p>
    </div>
  </div>
</template>

<script setup>
import { ref, onMounted } from 'vue'

const props = defineProps({
  userId: {
    type: Number,
    required: true
  }
})

const user = ref(null)
const loading = ref(true)
const error = ref(null)

// 使用async/await获取数据
async function fetchUserData() {
  loading.value = true
  error.value = null
  
  try {
    const response = await fetch(`https://api.example.com/users/${props.userId}`)
    if (!response.ok) {
      throw new Error(`API请求失败: ${response.status}`)
    }
    user.value = await response.json()
  } catch (err) {
    error.value = err.message
    console.error('获取用户数据失败:', err)
  } finally {
    loading.value = false
  }
}

// 当组件挂载时获取数据
onMounted(fetchUserData)

// 监听userId的变化重新获取数据
watch(() => props.userId, fetchUserData)
</script>

最佳实践

  1. 减少复杂度：将大型组件拆分为更小的可复用组件
  2. 使用组合函数：将逻辑提取到独立的组合函数（Composables）中
  3. 正确处理响应式：理解ref和reactive的区别
  4. 组件通信：适当使用props和emits，复杂场景考虑使用provide/inject
  5. 保持一致性：在项目中统一使用Setup语法糖或Options API

常见问题解答

问题1：<script setup> 中如何访问组件实例（this）？

答：在<script setup>中没有this。这是函数式编程的一部分，所有需要的内容都应通过导入的API和组合函数获取。

问题2：如何使用生命周期钩子？

答：直接导入并调用生命周期钩子函数：

<script setup>
import { onMounted, onUnmounted } from 'vue'

onMounted(() => {
  console.log('组件已挂载')
})

onUnmounted(() => {
  console.log('组件已卸载')
})
</script>

问题3：如何使用依赖注入（provide/inject）？

答：

<!-- 提供方 -->
<script setup>
import { provide, ref } from 'vue'

const theme = ref('light')
provide('theme', theme) // 提供响应式值
</script>

<!-- 注入方 -->
<script setup>
import { inject } from 'vue'

const theme = inject('theme', 'light') // 第二个参数是默认值
</script>

响应式系统

Vue 3的响应式系统是框架的核心特性，它允许数据变化自动触发视图更新。在Composition API中，响应式系统通过ref、reactive等API实现。

基本响应式API

  1. ref：创建一个响应式的数据引用，适用于基本类型和对象

    import { ref } from 'vue'

const count = ref(0)  // 基本类型
console.log(count.value)  // 必须使用.value访问

const user = ref({ name: 'Alice' })  // 对象
console.log(user.value.name)

  2. reactive：创建一个响应式对象，仅适用于对象（不能用于原始值）

    import { reactive } from 'vue'

const state = reactive({
  count: 0,
  user: { name: 'Bob' }
})
console.log(state.count)  // 直接访问，不需要.value

生命周期钩子

Vue 3中的生命周期钩子允许你在组件不同阶段执行代码。在Composition API中，生命周期钩子是以on开头的函数。

import { onMounted, onUpdated, onUnmounted } from 'vue'

// 在setup中使用生命周期钩子
onMounted(() => {
  console.log('组件已挂载')
})

onUpdated(() => {
  console.log('组件已更新')
})

onUnmounted(() => {
  console.log('组件已卸载')
})

ref和reactive的区别

  • ref定义基本数据类型、对象类型数据。reactive定义对象类型数据
  • ref创建的变量必须.value。reactive重新分配一个新对象会失去响应式，可以用Object.assign去整体替换
  • 定义层级比较深的响应式对象推荐用reactive
  • ref底层也是依赖reactive

计算属性computed()

<style>
    .person{
        background-color: #ddd;
    }
</style>

<template>
    <div class="person">
        输入姓：<input type="text" v-model="firstName"><br>
        输入名：<input type="text" v-model="lastName"><br>
        <span>普通方法调全名:{{fullName()}}</span><br>
        <span>普通方法调全名:{{fullName()}}</span><br>
        <span>计算属性调全名:{{fullNameComputed}}</span><br>
        <span>计算属性调全名:{{fullNameComputed}}</span><br>
        <button @click="updateFullname">计算属性修改全名:{{fullNameComputed}}</button>
    </div>
</template>

<script lang="ts" >
   export default{
        name:"Person123"
   }
</script>
<script lang="ts" setup >
    import { ref, computed } from 'vue'; //引入计算属性computed
    let firstName = ref("luo")
    let lastName = ref("tao")
    function fullName(){
        console.log(1); // 调用几次就打印几次，因为没有缓存
        return firstName.value.slice(0,1).toUpperCase() + firstName.value.slice(1) + lastName.value.slice(0,1).toUpperCase() + lastName.value.slice(1)
    }

    /*
    // 这么定义的fullNameComputed是只读的，可以通过set修改
    let fullNameComputed = computed(()=>{ 
        // 相当于java的override
        console.log("computed"); // 调用一次，computed会缓存
    })*/

    // 这么定义fullNameComputed可以通过set修改
    let fullNameComputed = computed({ 
        // 传递对象，相当于 java的setter
        get(){
            return firstName.value.slice(0,1).toUpperCase() + firstName.value.slice(1) + lastName.value.slice(0,1).toUpperCase() + lastName.value.slice(1)
        
        },
        set(val){
            const [firststr,laststr] = val.split("-")
            firstName.value=firststr
            lastName.value=laststr
        }
    })
    function updateFullname(){
        fullNameComputed.value="shuai-ge" // 传给set进行修改
    }
</script>

watch-监视【ref】定义的【基本类型】数据

  • 直接写数据名,监视的是value值的改变
<style>
    .person{
        background-color: #ddd;
    }
</style>

<template>
    <div class="person">
        <button @click="sum">{{count}}</button>
    </div>
</template>

<script lang="ts" >
   export default{
        name:"Person123"
   }
</script>
<script lang="ts" setup >
    import { ref, watch} from 'vue';
    let count =ref(0)
    function sum(){
        count.value +=1;
        console.log(count.value);
        
    }
    // watch 监视【ref】定义的【基本类型】数据
    const stopWach= watch(count,(newValue,oldValue)=>{
        console.log("count变化了",newValue,oldValue);
        if (newValue>=5){
            stopWach();
        }
    })
</script>

watch

  • 若修改的是ref定义的对象的属性，newValue和oldValue都是新值，因为它们是同一个对象,地址值不变。
  • 若修改整个ref定义的对象，newValue是新值，oldValue是旧值，因为不是同一个对象。监视的是地址值，若想监视对象内部属性的变化，需要开启深度监视。
