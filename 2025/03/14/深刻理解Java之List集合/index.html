深刻理解Java之List集合

发表于 2025-03-14

深刻理解Java之List集合

1. List接口概述

1.1 基本特性

List接口继承自Collection接口，代表一个有序的、可重复的元素集合。其主要特点：

  • 有序性：元素存储和添加的顺序一致
  • 可重复性：允许存储相同的元素
  • 索引访问：可以通过索引精确访问元素
  • 动态扩容：大多数实现类支持动态调整容量
  • 存储引用数据类型

生活比喻：List就像电影院的一排座位

  • 每个座位都有明确的编号(0, 1, 2…)，按顺序排列
  • 同一个人可以买多个座位(可重复)
  • 你可以直接找到”第5号座位”而不需要从头数过去
  • 如果需要，影院可以增加更多座位(动态扩容)

1.2 List在集合框架中的位置

Collection (接口)
├── List (接口)
│   ├── ArrayList (实现类)
│   ├── LinkedList (实现类)
│   ├── Vector (实现类)
│   │   └── Stack (子类)
│   └── ...
├── Set (接口)
└── Queue (接口)

1.3 List接口定义的主要方法

// 添加元素到指定位置
void add(int index, E element);

// 获取指定索引的元素
E get(int index);

// 替换指定索引的元素
E set(int index, E element);

// 移除指定索引的元素
E remove(int index);

// 查找元素首次出现的索引
int indexOf(Object o);

// 查找元素最后一次出现的索引
int lastIndexOf(Object o);

// 获取子列表
List<E> subList(int fromIndex, int toIndex);

2. List的主要实现类

2.1 ArrayList - “整齐排列的火车车厢”

ArrayList是List接口最常用的实现类，基于动态数组实现，查询快，但由于不可变性，增删会复制创建新数组。

内部结构：

// ArrayList的核心是一个Object数组
transient Object[] elementData;

// 实际元素数量
private int size;

工作原理：

  • 初始容量为10（默认构造函数）
  • 当容量不足时，自动扩容为原容量的1.5倍
  • 扩容涉及数组复制，是一个相对昂贵的操作

性能特点：

  • 随机访问性能极佳：O(1)
  • 尾部添加元素通常为O(1)，但可能触发扩容
  • 中间插入/删除元素性能较差：O(n)，因为需要移动元素
  • 非线程安全

生活比喻：
ArrayList就像一列火车车厢，整齐地排列在铁轨上：

  • 想找到第N号车厢非常容易，直接数过去就行（随机访问快）
  • 在中间加入新车厢很麻烦，需要把后面所有车厢都移动（中间插入慢）
  • 在尾部加车厢相对简单，但偶尔需要换一条更长的铁轨（扩容操作）
  • 每个车厢紧挨着，占用空间紧凑（内存效率高）

代码示例：

// 创建ArrayList
List<String> fruits = new ArrayList<>();

// 添加元素
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Orange");

// 指定位置添加
fruits.add(1, "Mango");  // [Apple, Mango, Banana, Orange]

// 访问元素
String fruit = fruits.get(0);  // Apple

// 修改元素
fruits.set(0, "Pineapple");  // [Pineapple, Mango, Banana, Orange]

// 删除元素
fruits.remove(2);  // [Pineapple, Mango, Orange]

// 查找元素
int index = fruits.indexOf("Mango");  // 1

// 大小
int siz = fruits.size();

2.2 LinkedList - “手拉手的人链”

LinkedList是基于双向链表实现的List接口，同时也实现了Deque接口。增删快，只需要改节点的指针，当时查询慢，需要从头开始找。

内部结构：

// 链表节点类
private static class Node<E> {
    E item;           // 元素值
    Node<E> next;     // 后继节点
    Node<E> prev;     // 前驱节点
}

// 链表大小
transient int size;

// 头节点和尾节点
transient Node<E> first;
transient Node<E> last;

工作原理：

  • 每个元素都是一个包含前后引用的节点
  • 添加元素时创建新节点并调整相关节点的引用
  • 删除元素时调整相关节点的引用

性能特点：

  • 随机访问性能较差：O(n)，需要从头/尾遍历
  • 插入/删除元素性能极佳：O(1)，只需调整引用
  • 内存占用较大，每个元素需要额外存储前后引用
  • 非线程安全

生活比喻：
LinkedList就像一群手拉手的人形成的人链：

  • 要找到第50个人，你必须从头（或尾）数过去（随机访问慢）
  • 在任何位置插入新人很容易，只需要松开两个人的手，让新人拉住他们（插入快）
  • 每个人除了自己外，还要记住前后两个人是谁（额外内存开销）
  • 人链的两端可以很方便地加入或移除人员（两端操作高效）

代码示例：

// 创建LinkedList
LinkedList<String> tasks = new LinkedList<>();

// 添加元素
tasks.add("Task 1");
tasks.add("Task 3");

// 在指定位置添加
tasks.add(1, "Task 2");  // [Task 1, Task 2, Task 3]

// 作为双端队列使用
tasks.addFirst("Task 0");  // [Task 0, Task 1, Task 2, Task 3]
tasks.addLast("Task 4");   // [Task 0, Task 1, Task 2, Task 3, Task 4]

// 获取元素
String first = tasks.getFirst();  // Task 0
String last = tasks.getLast();    // Task 4

// 删除元素
tasks.removeFirst();  // [Task 1, Task 2, Task 3, Task 4]
tasks.removeLast();   // [Task 1, Task 2, Task 3]

2.3 Vector - “带锁的保险柜”

Vector是List接口的古老实现类，与ArrayList类似，但是线程安全的。

内部结构：

  • 与ArrayList类似，基于数组实现
  • 方法使用synchronized关键字同步

性能特点：

  • 随机访问性能好：O(1)
  • 由于同步机制，性能比ArrayList差
  • 扩容时默认增长为原容量的2倍（而ArrayList是1.5倍）
  • 线程安全

生活比喻：
Vector就像一个带锁的保险柜，每次只允许一个人操作：

  • 查找和存取物品的方式与普通柜子相同
  • 但每次操作都需要先开锁，用完再锁上（同步操作的开销）
  • 安全性高，但效率低

适用场景：

  • 需要线程安全的动态数组
  • 多线程环境下共享集合
  • 注：现代Java应用中，通常使用Collections.synchronizedList()或CopyOnWriteArrayList代替Vector

2.4 Stack - “书堆”

Stack是Vector的子类，实现了后进先出(LIFO)的栈结构。

主要方法：

  • push(E item)：将元素压入栈顶
  • pop()：移除并返回栈顶元素
  • peek()：返回栈顶元素但不移除
  • empty()：检查栈是否为空
  • search(Object o)：查找元素在栈中的位置

生活比喻：
Stack就像一堆叠放的书：

  • 你只能从顶部放书或取书
  • 要拿到底部的书，必须先移除上面所有的书

注意：Stack类已被视为过时设计，现代Java应用通常使用Deque接口的实现（如ArrayDeque）作为栈。

4. List的常用操作

4.1 创建List

// 空ArrayList
List<String> list1 = new ArrayList<>();

// 指定初始容量
List<String> list2 = new ArrayList<>(20);

// 基于已有集合创建
List<String> list3 = new ArrayList<>(anotherCollection);

// 使用Arrays.asList创建固定大小的List
List<String> list4 = Arrays.asList("A", "B", "C");
// 注意：Arrays.asList返回的List不支持add/remove操作

// Java 9+: 使用List.of创建不可变List
List<String> list5 = List.of("A", "B", "C");

// 使用Stream创建
List<Integer> list6 = IntStream.range(1, 6)
                              .boxed()
                              .collect(Collectors.toList());

生活比喻：

  • 创建空ArrayList就像买了一个空相册，准备放照片
  • 指定初始容量就像提前准备好20个格子的收纳盒
  • Arrays.asList就像买了一个已经放了照片的相册，但相册大小固定

4.2 添加元素

List<String> colors = new ArrayList<>();

// 添加单个元素
colors.add("Red");

// 在指定位置添加
colors.add(0, "Blue");

// 添加多个元素
colors.addAll(Arrays.asList("Green", "Yellow"));

// 在指定位置添加集合
colors.addAll(1, Arrays.asList("Purple", "Orange"));

4.3 访问元素

List<String> planets = new ArrayList<>(
    Arrays.asList("Mercury", "Venus", "Earth", "Mars", "Jupiter")
);

// 通过索引访问
String earth = planets.get(2);  // Earth

// 查找元素位置
int index = planets.indexOf("Mars");  // 3
int lastIndex = planets.lastIndexOf("Earth");  // 2

// 检查是否包含元素
boolean contains = planets.contains("Venus");  // true

// 获取子列表（视图，非副本）
List<String> innerPlanets = planets.subList(0, 4);

4.4 修改元素

List<String> names = new ArrayList<>(
    Arrays.asList("Alice", "Bob", "Charlie", "David")
);

// 替换元素
names.set(1, "Robert");  // [Alice, Robert, Charlie, David]

// 批量替换（Java 8+）
names.replaceAll(name -> name.toUpperCase());
// [ALICE, ROBERT, CHARLIE, DAVID]

4.5 删除元素

List<String> animals = new ArrayList<>(
    Arrays.asList("Dog", "Cat", "Bird", "Fish", "Cat")
);

// 删除指定位置的元素
String removed = animals.remove(2);  // Bird
// 现在是 [Dog, Cat, Fish, Cat]

// 删除指定元素（首次出现）
boolean isRemoved = animals.remove("Cat");  // true
// 现在是 [Dog, Fish, Cat]

// 批量删除
animals.removeAll(Arrays.asList("Fish", "Cat"));
// 现在是 [Dog]

// 条件删除（Java 8+）
animals.add("Elephant");
animals.add("Snake");
animals.removeIf(animal -> animal.length() > 3);
// 现在是 [Dog]

4.6 遍历List

iterator方法 说明
hasNext() 判断集合中是否还有下一个元素
next() 返回集合中的下一个元素
remove() 删除迭代器返回的元素 
List<String> fruits = Arrays.asList("Apple", "Banana", "Cherry", "Date");

// 1. 使用for循环（索引）
for (int i = 0; i < fruits.size(); i++) {
    System.out.println(fruits.get(i));
}

// 2. 使用增强for循环
for (String fruit : fruits) {
    System.out.println(fruit);
}

// 3. 使用迭代器时，集合中的元素删除只能通过迭代器提供remove方法
Iterator<String> iterator = fruits.iterator();
while (iterator.hasNext()) {
    String f = iterator.next()
    if(f=="Banana"){
        iterator.remove();
    }
}

// 4. 使用ListIterator（可双向遍历）
ListIterator<String> listIterator = fruits.listIterator();
while (listIterator.hasNext()) {
    System.out.println(listIterator.next());
}
// 反向遍历
while (listIterator.hasPrevious()) {
    System.out.println(listIterator.previous());
}

// 5. 使用forEach方法（Java 8+）
fruits.forEach(System.out::println);

// 6. 使用Stream API（Java 8+）
fruits.stream()
      .filter(f -> f.startsWith("A"))
      .map(String::toUpperCase)
      .forEach(System.out::println);

生活比喻：

  • for循环（索引）：就像按座位号点名
  • 增强for循环：就像派发礼物，每个人都要领到，但不关心座位号
  • 迭代器：就像列队检查，检查完一个人后，才移动到下一个人
  • Stream API：就像设置了一条传送带，水果经过清洗(filter)、切片(map)等处理

4.7 排序List

List<String> names = new ArrayList<>(
    Arrays.asList("Zack", "Alice", "Bob", "Charlie")
);

// 1. 使用Collections.sort（自然顺序）
Collections.sort(names);
// [Alice, Bob, Charlie, Zack]

// 2. 使用List.sort（Java 8+）
names.sort(Comparator.naturalOrder());
// [Alice, Bob, Charlie, Zack]

// 3. 自定义比较器
names.sort(Comparator.comparing(String::length));
// 按长度排序: [Bob, Zack, Alice, Charlie]

// 4. 反向排序
names.sort(Comparator.reverseOrder());
// [Zack, Charlie, Bob, Alice]

// 5. 复合排序
names.sort(
    Comparator.comparing(String::length)
              .thenComparing(Comparator.naturalOrder())
);
// 先按长度，再按字母顺序

生活比喻：

  • 自然顺序排序就像按字母表顺序排队
  • 自定义比较器就像可以选择按年龄、身高或成绩给学生排队
  • 复合排序就像先按年级分组，再按成绩排序

5. List的线程安全性

5.1 线程不安全的List

ArrayList和LinkedList都不是线程安全的，在多线程环境下可能出现：

  • ConcurrentModificationException
  • 数据不一致
  • 元素丢失

生活比喻：
就像没有管理员的图书馆书架，多人同时取放书可能导致混乱（有人正在整理的书架被其他人打乱）

5.2 创建线程安全的List

// 1. 使用Collections.synchronizedList
List<String> syncList = Collections.synchronizedList(new ArrayList<>());

// 使用时需要手动同步迭代操作
synchronized (syncList) {
    Iterator<String> iterator = syncList.iterator();
    while (iterator.hasNext()) {
        // ...
    }
}

// 2. 使用CopyOnWriteArrayList（适合读多写少的场景）
List<String> cowList = new CopyOnWriteArrayList<>();
// 写操作会复制整个底层数组，读操作无锁
// 迭代器是快照，不会抛出ConcurrentModificationException

生活比喻：

  • synchronizedList就像有管理员的图书馆，一次只允许一个人操作书架，其他人必须等待
  • CopyOnWriteArrayList就像特殊的图书馆，每次有人要修改书架，都会先复制一份完整的书架进行修改，读者始终看到的是完整的旧版本，直到修改完成

6. List的性能优化

6.1 ArrayList优化

// 1. 指定初始容量，避免频繁扩容
List<String> list = new ArrayList<>(10000);

// 2. 批量添加元素
list.addAll(collection);  // 优于循环添加

// 3. 使用ensureCapacity预分配空间
ArrayList<String> arrayList = new ArrayList<>();
arrayList.ensureCapacity(10000);

// 4. 使用trimToSize释放多余空间
arrayList.trimToSize();

生活比喻：

  • 指定初始容量就像搬家时一开始就租足够大的卡车，比先租小车再换大车要高效得多
  • trimToSize就像搬完家后退掉大卡车，换成刚好够用的小车，节省空间

6.2 LinkedList优化

// 1. 使用合适的方法
LinkedList<String> linkedList = new LinkedList<>();
// 使用addFirst/addLast而非add(0, element)
linkedList.addFirst("First");  // 比add(0, "First")更高效

// 2. 避免随机访问
// 不要使用for (int i = 0; i < list.size(); i++)
// 而应使用迭代器或增强for循环

生活比喻：

  • 使用addFirst而非add(0, element)就像直接告诉门卫”我要见队伍最前面的人”，而不是让门卫带你从队伍头走到尾

7. List的实际应用场景

7.1 ArrayList应用场景

  • 实现动态数组
  • 频繁随机访问的数据集合
  • 数据量较大但变化较少的集合
  • 需要快速迭代的场景

生活比喻：
ArrayList就像一本书的目录或索引，需要快速查找但不经常修改

7.2 LinkedList应用场景

  • 实现队列或栈
  • 频繁插入/删除操作的场景
  • 实现LRU缓存
  • 需要从两端操作的集合

生活比喻：
LinkedList就像餐厅的等位名单，经常有人加入或离开队伍

7.3 实际案例：分页处理

public class Pagination<T> {
    private final List<T> data;
    private final int pageSize;
    
    public Pagination(List<T> data, int pageSize) {
        this.data = new ArrayList<>(data);  // 创建副本，避免外部修改
        this.pageSize = pageSize;
    }
    
    public List<T> getPage(int pageNumber) {
        int startIndex = (pageNumber - 1) * pageSize;
        int endIndex = Math.min(startIndex + pageSize, data.size());
        
        if (startIndex >= data.size() || startIndex < 0) {
            return Collections.emptyList();
        }
        
        return data.subList(startIndex, endIndex);
    }
    
    public int getTotalPages() {
        return (int) Math.ceil((double) data.size() / pageSize);
    }
}

// 使用示例
List<Product> allProducts = productRepository.findAll();
Pagination<Product> pagination = new Pagination<>(allProducts, 10);
List<Product> page1 = pagination.getPage(1);  // 获取第一页

生活比喻：
分页处理就像一本厚书的章节划分，让读者能够一次阅读适量的内容，而不是一次展示所有内容

8. List的最佳实践

  1. 选择合适的实现类

    • 随机访问多，选择ArrayList
    • 插入/删除多，选择LinkedList
    • 需要线程安全，使用CopyOnWriteArrayList或synchronizedList

    比喻：就像选择交通工具 - 需要在固定站点上下客，选择公交车(ArrayList)；需要灵活穿梭于各个小巷，选择摩托车(LinkedList)

  2. 避免频繁扩容

    • 预估容量，使用带容量参数的构造函数
    • 批量添加元素时使用addAll而非循环add

    比喻：一开始就买足够大的水桶，而不是先买小桶再换大桶

  3. 正确使用迭代器

    • 使用迭代器遍历时不要使用集合的remove方法，应使用迭代器的remove方法
      // 错误方式
for (String item : list) {
    if (condition) list.remove(item);  // ConcurrentModificationException
}

// 正确方式
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    String item = it.next();
    if (condition) it.remove();
}

    比喻：就像在移动的传送带上拿东西 - 直接改变传送带的结构会导致故障，应该通过传送带提供的控制按钮来移除物品

  4. 合理使用视图和包装器

    • subList返回的是原List的视图，修改会影响原List
    • unmodifiableList创建只读视图，防止修改

    比喻：subList就像房间的窗户，通过窗户看到的变化反映了房间的实际变化；unmodifiableList就像博物馆的展品，只能看不能动

  5. 注意List.of和Arrays.asList的限制

    • List.of和Arrays.asList创建的List不支持添加/删除操作
    • Arrays.asList创建的List大小固定，但元素可修改
    • List.of创建的List完全不可变

    比喻：Arrays.asList就像固定格数的相册，可以替换照片但不能增减格子；List.of就像密封的时间胶囊，一旦封存就不能改变

  6. 利用Stream API进行复杂操作

    • 使用Stream API进行过滤、映射、归约等操作
      List<Integer> evenNumbers = numbers.stream()
                                  .filter(n -> n % 2 == 0)
                                  .collect(Collectors.toList());

    比喻：Stream API就像工厂的生产线，原材料(数据)经过一系列工序(操作)，最终成为成品

  7. 避免装箱/拆箱开销

    • 对于基本类型，考虑使用专门的实现如IntArrayList（第三方库）
    • 或使用数组代替List

    比喻：装箱/拆箱就像每次进出商场都要把东西包装/拆包，很耗时间

  8. 注意equals和hashCode

    • 自定义类作为List元素时，正确实现equals和hashCode方法
    • 这对contains、remove等方法的正确工作至关重要

    比喻：equals就像人脸识别系统，如果设计不当，可能会把双胞胎认成同一个人或把同一个人的不同表情认成不同人
