深刻理解Java之Map集合

什么是Map集合

Map集合是Java集合框架中一个重要的接口，它用于存储键值对（key-value pairs）。想象一个图书馆的书架系统：每本书都有一个唯一的编号（key），通过这个编号可以找到对应的书（value）。这就是Map的基本工作原理。

生活中的例子：

  • 学生证号 -> 学生信息
  • 身份证号 -> 居民信息
  • 手机号码 -> 联系人
  • 银行卡号 -> 账户信息

在Java中，Map接口位于java.util包中，它的基本定义如下：

public interface Map<K,V> {
    // K 表示键（Key）的类型
    // V 表示值（Value）的类型
}

Map接口的特点

  1. 键值对映射：每个元素都是键值对，包含一个键和一个值
  2. 键的唯一性：键不能重复，但值可以重复
  3. 键值对的存储：键和值都可以是任何引用类型的数据，但一般不建议使用可变对象作为键
  4. 键和null：大多数Map实现允许使用null键（TreeMap除外），但只能有一个null键
  5. 值和null：所有Map实现都允许使用null值，且可以有多个null值

Map接口的主要实现类

HashMap

HashMap是最常用的Map实现类，就像一个超市的储物柜系统：每个柜子都有一个唯一的号码（key），用来存储顾客的物品（value）。

特点：

  • 无序存储
  • 允许null键和null值
  • 非同步（线程不安全）
  • 查找和插入的时间复杂度为O(1)

内部实现：

public class HashMap<K,V> extends AbstractMap<K,V> implements Map<K,V> {
    // 默认初始容量 - 必须是2的幂
    static final int DEFAULT_INITIAL_CAPACITY = 16;
    
    // 默认加载因子
    static final float DEFAULT_LOAD_FACTOR = 0.75f;
    
    // 当链表长度大于此值时，转换为红黑树
    static final int TREEIFY_THRESHOLD = 8;
    
    // Node数组，用于存储键值对
    transient Node<K,V>[] table;
}

LinkedHashMap

LinkedHashMap是HashMap的子类，它在HashMap的基础上维护了一个双向链表，用于记录元素的插入顺序或访问顺序。想象一个有序的超市储物柜，不仅能通过柜号找到物品，还能按存入顺序或使用频率排序。

特点：

  • 维护元素的插入顺序或访问顺序（可选）
  • 适合实现LRU缓存
  • 性能略低于HashMap
  • 允许null键和null值

示例：

// 创建一个记录访问顺序的LinkedHashMap（LRU缓存实现）
LinkedHashMap<String, String> lruCache = new LinkedHashMap<>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, String> eldest) {
        return size() > 3; // 限制缓存大小为3
    }
};

TreeMap

TreeMap基于红黑树实现，它的键值对总是按照键的自然顺序或指定的比较器排序。想象一个按字母顺序排列的字典，你可以快速找到特定字母开头的词条。

特点：

  • 键值对按键排序（自然顺序或自定义顺序）
  • 不允许null键，但允许null值
  • 查找和插入的时间复杂度为O(log n)
  • 适合需要排序的场景

示例：

// 创建一个按键的长度排序的TreeMap
TreeMap<String, Integer> lengthSortedMap = new TreeMap<>((s1, s2) -> {
    int lenCompare = Integer.compare(s1.length(), s2.length());
    return lenCompare != 0 ? lenCompare : s1.compareTo(s2);
});

自然排序

package com.luotao.job.utils.demo;

import java.util.Map;
import java.util.TreeMap;

// 实现Comparable接口的类
class Book implements Comparable<Book> {
    private String title;
    private double price;

    public Book(String title, double price) {
        this.title = title;
        this.price = price;
    }

    @Override
    public int compareTo(Book other) {
        // 按书名字母顺序排序
        return this.title.compareTo(other.title);
    }

    @Override
    public String toString() {
        return "Book{title='" + title + "', price=" + price + "}";
    }
}

 class ComparableDemo {
    public static void main(String[] args) {
        TreeMap<Book, Integer> bsMap = new TreeMap<>();

        bsMap.put(new Book("1Java编程", 59.9),1);
        bsMap.put(new Book("3Python基础", 45.5),2);
        bsMap.put(new Book("2算法导论", 88.0),3);
        bsMap.put(new Book("4数据结构", 55.0),4);

        // 遍历（按书名字母顺序）
        for (Map.Entry entry: bsMap.entrySet()){
            System.out.println(entry.getKey() + ":" + entry.getValue());
        }
    }
}

Hashtable

Hashtable是一个历史悠久的Map实现，现在主要用于遗留代码。它的所有方法都是同步的，类似于一个带锁的储物柜系统。

特点：

  • 线程安全（所有方法都是同步的）
  • 不允许null键和null值
  • 性能较差（因为同步）
  • 已被ConcurrentHashMap取代

ConcurrentHashMap

ConcurrentHashMap是专门为并发设计的Map实现，它像一个分区的储物柜系统，不同的区域可以同时被不同的人使用。

特点：

  • 线程安全
  • 支持高并发访问
  • 不允许null键和null值
  • 性能优于Hashtable

示例：

ConcurrentHashMap<String, Integer> concurrentMap = new ConcurrentHashMap<>();
// 线程安全的操作
concurrentMap.computeIfAbsent("count", k -> 0);
concurrentMap.computeIfPresent("count", (k, v) -> v + 1);

WeakHashMap

WeakHashMap的特殊之处在于它的键是弱引用，当键对象不再被其他对象引用时，相应的键值对会被自动移除。想象一个会自动清理的储物柜系统。

特点：

  • 键是弱引用
  • 适合实现缓存
  • 自动移除无用的键值对
  • 需要注意垃圾回收的影响

EnumMap

EnumMap是专门为枚举类型设计的Map实现，非常高效。

特点：

  • 键必须是同一个枚举类型
  • 不允许null键
  • 内部使用数组实现，性能极高
  • 保持枚举定义的顺序

Map集合的遍历方式

1. 使用增强for循环遍历键值对

Map<String, Integer> map = new HashMap<>();
map.put("Apple", 1);
map.put("Banana", 2);
map.put("Orange", 3);

// 遍历键值对
for (Map.Entry<String, Integer> entry : map.entrySet()) {
    System.out.println("Key: " + entry.getKey() + ", Value: " + entry.getValue());
}

// 只遍历键
for (String key : map.keySet()) {
    System.out.println("Key: " + key);
}

// 只遍历值
for (Integer value : map.values()) {
    System.out.println("Value: " + value);
}

2. 使用Lambda表达式遍历

Map<String, Integer> map = new HashMap<>();
map.put("Apple", 1);
map.put("Banana", 2);
map.put("Orange", 3);

// 使用forEach方法
map.forEach((key, value) -> {
    System.out.println("Key: " + key + ", Value: " + value);
});

3. 使用Stream API遍历

Map<String, Integer> map = new HashMap<>();
map.put("Apple", 1);
map.put("Banana", 2);
map.put("Orange", 3);

// 使用Stream API处理键值对
map.entrySet().stream()
   .filter(entry -> entry.getValue() > 1)
   .forEach(entry -> System.out.println(entry.getKey() + ": " + entry.getValue()));

Map集合的应用场景

  1. 缓存实现：使用Map存储计算结果，避免重复计算。

    Map<String, Object> cache = new LinkedHashMap<>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, Object> eldest) {
        return size() > 100; // 限制缓存大小
    }
};

  2. 计数器：统计元素出现的次数。

    Map<String, Integer> wordCount = new HashMap<>();
String[] words = {"apple", "banana", "apple", "orange"};
for (String word : words) {
    wordCount.merge(word, 1, Integer::sum);
}

  3. 数据索引：建立快速查找索引。

    Map<String, User> userIndex = new HashMap<>();
List<User> users = getUserList();
users.forEach(user -> userIndex.put(user.getId(), user));

  4. 配置管理：存储配置项和对应的值。

    Map<String, String> config = new Properties();
config.put("db.url", "jdbc:mysql://localhost:3306/mydb");
config.put("db.username", "root");

  5. 多值映射：一个键对应多个值。

    Map<String, List<String>> multiMap = new HashMap<>();
multiMap.computeIfAbsent("fruits", k -> new ArrayList<>()).add("apple");
multiMap.computeIfAbsent("fruits", k -> new ArrayList<>()).add("banana");

常见问题与最佳实践

1. 如何选择合适的Map实现？

  • 一般场景：使用HashMap
  • 需要保持插入顺序：使用LinkedHashMap
  • 需要键排序：使用TreeMap
  • 需要线程安全：使用ConcurrentHashMap
  • 键是枚举类型：使用EnumMap
  • 需要缓存功能：使用WeakHashMap或自定义的LinkedHashMap

2. 自定义对象作为键的注意事项

public class CustomKey {
    private String id;
    private String name;
    
    // 必须正确实现equals和hashCode方法
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        CustomKey customKey = (CustomKey) o;
        return Objects.equals(id, customKey.id) &&
               Objects.equals(name, customKey.name);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(id, name);
    }
}

3. 避免并发修改问题

Map<String, Integer> map = new HashMap<>();
// 错误的方式
for (String key : map.keySet()) {
    if (someCondition) {
        map.remove(key); // 可能抛出ConcurrentModificationException
    }
}

// 正确的方式
Iterator<Map.Entry<String, Integer>> iterator = map.entrySet().iterator();
while (iterator.hasNext()) {
    Map.Entry<String, Integer> entry = iterator.next();
    if (someCondition) {
        iterator.remove(); // 安全的删除方式
    }
}

Map<String, Integer> map = new HashMap<>();

map.compute("key", (k, v) -> (v == null) ? 1 : v + 1);
map.computeIfAbsent("key", k -> 1);
map.computeIfPresent("key", (k, v) -> v + 1);

// 使用合并方法
map.merge("key", 1, Integer::sum);

// 使用getOrDefault

代码示例

示例1：基本操作

import java.util.*;

public class MapDemo {
    public static void main(String[] args) {
        // 创建HashMap
        Map<String, Integer> scores = new HashMap<>();
        
        // 添加元素
        scores.put("Alice", 95);
        scores.put("Bob", 85);
        scores.put("Charlie", 90);
        
        // 获取元素
        System.out.println("Bob's score: " + scores.get("Bob"));
        
        // 使用getOrDefault
        System.out.println("David's score: " + scores.getOrDefault("David", 0));
        
        // 检查键是否存在
        System.out.println("Has Alice? " + scores.containsKey("Alice"));
        System.out.println("Has score 85? " + scores.containsValue(85));
        
        // 更新值
        scores.put("Bob", 88); // 覆盖原值
        
        // 只在键不存在时添加
        scores.putIfAbsent("Bob", 100); // 不会改变Bob的分数
        
        // 删除元素
        scores.remove("Charlie");
        
        // 遍历Map
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.printf("%s scored %d%n", entry.getKey(), entry.getValue());
        }
        
        // 清空Map
        scores.clear();
        System.out.println("Is empty? " + scores.isEmpty());
    }
}

示例2：不同Map实现的比较

import java.util.*;

public class MapComparison {
    public static void main(String[] args) {
        // HashMap - 无序
        Map<String, Integer> hashMap = new HashMap<>();
        hashMap.put("C", 3);
        hashMap.put("A", 1);
        hashMap.put("B", 2);
        System.out.println("HashMap: " + hashMap);
        
        // LinkedHashMap - 保持插入顺序
        Map<String, Integer> linkedHashMap = new LinkedHashMap<>();
        linkedHashMap.put("C", 3);
        linkedHashMap.put("A", 1);
        linkedHashMap.put("B", 2);
        System.out.println("LinkedHashMap: " + linkedHashMap);
        
        // TreeMap - 键排序
        Map<String, Integer> treeMap = new TreeMap<>();
        treeMap.put("C", 3);
        treeMap.put("A", 1);
        treeMap.put("B", 2);
        System.out.println("TreeMap: " + treeMap);
        
        // ConcurrentHashMap - 线程安全
        Map<String, Integer> concurrentMap = new ConcurrentHashMap<>();
        concurrentMap.put("C", 3);
        concurrentMap.put("A", 1);
        concurrentMap.put("B", 2);
        System.out.println("ConcurrentHashMap: " + concurrentMap);
    }
}

示例3：Map的高级操作

import java.util.*;

public class AdvancedMapOperations {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();
        
        // 使用compute方法
        map.compute("A", (k, v) -> (v == null) ? 1 : v + 1);
        map.compute("A", (k, v) -> v + 1);
        System.out.println("A count: " + map.get("A")); // 输出2
        
        // 使用computeIfAbsent
        map.computeIfAbsent("B", k -> 1);
        map.computeIfAbsent("B", k -> 2); // 不会改变B的值
        System.out.println("B count: " + map.get("B")); // 输出1
        
        // 使用computeIfPresent
        map.computeIfPresent("B", (k, v) -> v + 1);
        System.out.println("B count: " + map.get("B")); // 输出2
        
        // 使用merge方法
        map.merge("C", 1, Integer::sum);
        map.merge("C", 1, Integer::sum);
        System.out.println("C count: " + map.get("C")); // 输出2
        
        // 批量操作
        map.replaceAll((k, v) -> v * 2);
        System.out.println("After doubling all values: " + map);
        
        // 使用Stream API
        int sum = map.values().stream()
                    .mapToInt(Integer::intValue)
                    .sum();
        System.out.println("Sum of all values: " + sum);
        
        // 按条件过滤
        Map<String, Integer> filteredMap = new HashMap<>();
        map.forEach((k, v) -> {
            if (v > 2) {
                filteredMap.put(k, v);
            }
        });
        System.out.println("Filtered map: " + filteredMap);
    }
}

示例5：实现LRU缓存

import java.util.*;

public class LRUCache<K, V> extends LinkedHashMap<K, V> {
    private final int capacity;
    
    public LRUCache(int capacity) {
        super(capacity, 0.75f, true);
        this.capacity = capacity;
    }
    
    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        return size() > capacity;
    }
    
    public static void main(String[] args) {
        LRUCache<String, Integer> cache = new LRUCache<>(3);
        
        cache.put("A", 1);
        cache.put("B", 2);
        cache.put("C", 3);
        System.out.println("Cache: " + cache); // {A=1, B=2, C=3}
        
        cache.get("A"); // 访问A，A移动到最后
        cache.put("D", 4); // 添加D，B将被移除（最久未使用）
        System.out.println("Cache after adding D: " + cache); // {C=3, A=1, D=4}
        
        cache.put("E", 5); // 添加E，C将被移除
        System.out.println("Cache after adding E: " + cache); // {A=1, D=4, E=5}
    }
}
