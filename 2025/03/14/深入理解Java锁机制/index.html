深入理解Java锁机制

发表于 2025-03-14

Java锁机制深入理解

一、锁的基本概念

1. 什么是锁？

专业概念：
锁是一种同步机制，用于控制多个线程对共享资源的访问。它能够保证在同一时刻只有一个线程可以访问共享资源，从而避免并发问题。

生活化比喻：

  • 就像”厕所门锁”，确保同一时间只有一个人能使用
  • 类似于”银行金库”，需要钥匙才能进入
  • 像是一个”会议室”，需要预约才能使用

2. 锁的分类

2.1 乐观锁与悲观锁

专业概念：

  • 乐观锁：假设不会发生冲突，只在更新时检查是否被修改
  • 悲观锁：假设会发生冲突，每次访问都加锁

代码示例：

// 乐观锁示例
public class OptimisticLock {
    private int version;
    private int value;
    
    public boolean update(int newValue) {
        // 假设不会冲突，直接更新
        this.value = newValue;
        this.version++;
        return true;
    }
}

// 悲观锁示例
public class PessimisticLock {
    private int value;
    
    public synchronized void update(int newValue) {
        // 假设会冲突，加锁保护
        this.value = newValue;
    }
}

生活化比喻：

  • 乐观锁就像”图书馆借书”，假设不会有人同时借同一本书
  • 悲观锁就像”银行取款”，每次都要排队等待

2.2 公平锁与非公平锁

专业概念：

  • 公平锁：按照申请锁的顺序获取锁
  • 非公平锁：允许插队，性能更好

代码示例：

// 公平锁
ReentrantLock fairLock = new ReentrantLock(true);

// 非公平锁
ReentrantLock unfairLock = new ReentrantLock(false);

生活化比喻：

  • 公平锁就像”排队买票”，先来先得
  • 非公平锁就像”公交车”，谁先挤上去谁先走

2.3 可重入锁与不可重入锁

专业概念：

  • 可重入锁：同一线程可以多次获取同一把锁
  • 不可重入锁：同一线程不能多次获取同一把锁

代码示例：

// 可重入锁示例
public class ReentrantLockDemo {
    private ReentrantLock lock = new ReentrantLock();
    
    public void outer() {
        lock.lock();  // 第一次获取锁
        try {
            inner();  // 可以再次获取锁
        } finally {
            lock.unlock();
        }
    }
    
    public void inner() {
        lock.lock();  // 第二次获取锁
        try {
            // 做一些事情
        } finally {
            lock.unlock();
        }
    }
}

生活化比喻：

  • 可重入锁就像”会员卡”，可以重复使用
  • 不可重入锁就像”一次性门票”，使用后就不能再用

二、Java中的锁实现

1. synchronized关键字

专业概念：
synchronized是Java内置的锁机制，可以修饰方法或代码块，保证线程安全。

代码示例：

public class SynchronizedDemo {
    // 修饰实例方法
    public synchronized void method1() {
        // 同步代码
    }
    
    // 修饰静态方法
    public static synchronized void method2() {
        // 同步代码
    }
    
    // 修饰代码块
    public void method3() {
        synchronized(this) {
            // 同步代码
        }
    }
}

生活化比喻：

  • 就像”会议室预约系统”，同一时间只能有一个团队使用
  • 类似于”图书馆座位”，需要刷卡才能使用

2. ReentrantLock

专业概念：
ReentrantLock是Java提供的可重入锁实现，比synchronized更灵活。

代码示例：

public class ReentrantLockDemo {
    private ReentrantLock lock = new ReentrantLock();
    
    public void method() {
        lock.lock();  // 获取锁
        try {
            // 临界区代码
        } finally {
            lock.unlock();  // 释放锁
        }
    }
    
    // 带超时的锁
    public boolean tryLockWithTimeout() {
        try {
            return lock.tryLock(5, TimeUnit.SECONDS);
        } catch (InterruptedException e) {
            return false;
        }
    }
}

生活化比喻：

  • 就像”智能门锁”，可以设置密码、指纹等多种方式
  • 类似于”预约系统”，可以设置超时时间

3. ReadWriteLock

专业概念：
读写锁允许多个线程同时读取，但写操作需要独占锁。

代码示例：

public class ReadWriteLockDemo {
    private ReadWriteLock rwLock = new ReentrantReadWriteLock();
    private Map<String, String> cache = new HashMap<>();
    
    public String read(String key) {
        rwLock.readLock().lock();
        try {
            return cache.get(key);
        } finally {
            rwLock.readLock().unlock();
        }
    }
    
    public void write(String key, String value) {
        rwLock.writeLock().lock();
        try {
            cache.put(key, value);
        } finally {
            rwLock.writeLock().unlock();
        }
    }
}

生活化比喻：

  • 就像”图书馆”，可以同时有多个人看书，但只能有一个人整理书架
  • 类似于”共享文档”，可以同时有多个人查看，但只能有一个人编辑

三、锁的性能优化

1. 锁的粒度

专业概念：
锁的粒度是指锁定的范围大小，粒度越小，并发性能越好。

代码示例：

// 粗粒度锁
public class CoarseGrainedLock {
    private synchronized void process() {
        // 处理大量数据
    }
}

// 细粒度锁
public class FineGrainedLock {
    private void process() {
        synchronized(part1) {
            // 处理部分数据
        }
        synchronized(part2) {
            // 处理另一部分数据
        }
    }
}

生活化比喻：

  • 粗粒度锁就像”整栋楼的门禁”，进出都要刷卡
  • 细粒度锁就像”每个房间的门锁”，可以同时使用不同房间

2. 锁分离

专业概念：
将读写操作分离，使用不同的锁，提高并发性能。

代码示例：

public class LockSeparation {
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    private final Lock readLock = rwLock.readLock();
    private final Lock writeLock = rwLock.writeLock();
    
    public void read() {
        readLock.lock();
        try {
            // 读取操作
        } finally {
            readLock.unlock();
        }
    }
    
    public void write() {
        writeLock.lock();
        try {
            // 写入操作
        } finally {
            writeLock.unlock();
        }
    }
}

生活化比喻：

  • 就像”图书馆”，读者和图书管理员使用不同的通道
  • 类似于”银行”，客户和工作人员使用不同的窗口

3. 锁消除

专业概念：
JVM在运行时，对一些不可能存在共享资源竞争的锁进行消除。

代码示例：

public class LockElimination {
    public String concatString(String s1, String s2, String s3) {
        return s1 + s2 + s3;  // JVM会消除这里的锁
    }
}

生活化比喻：

  • 就像”单人办公室”，不需要上锁
  • 类似于”私人储物柜”，不需要密码

四、常见面试题

1. synchronized和ReentrantLock的区别

问题： synchronized和ReentrantLock有什么区别？

详细答案：

  1. 使用方式

    • synchronized是关键字，使用更简单
    • ReentrantLock是类，需要手动加锁解锁

  2. 功能特性

    • ReentrantLock可以尝试获取锁
    • ReentrantLock可以响应中断
    • ReentrantLock可以实现公平锁

  3. 性能差异

    • JDK1.6后synchronized做了很多优化
    • 两者性能基本持平

生活化比喻：

  • synchronized就像”自动门”，使用简单
  • ReentrantLock就像”智能门锁”，功能更多

2. 死锁问题

问题： 什么是死锁？如何避免死锁？

详细答案：

  1. 死锁定义

    • 多个线程互相持有对方需要的锁
    • 导致所有线程都无法继续执行

  2. 避免方法

    • 固定加锁顺序
    • 使用超时机制
    • 死锁检测

代码示例：

public class DeadLockDemo {
    private Object lock1 = new Object();
    private Object lock2 = new Object();
    
    public void method1() {
        synchronized(lock1) {
            synchronized(lock2) {
                // 业务逻辑
            }
        }
    }
    
    public void method2() {
        synchronized(lock2) {
            synchronized(lock1) {
                // 业务逻辑
            }
        }
    }
}

生活化比喻：

  • 就像”两个人互相等待对方让路”
  • 类似于”两个人都想先过独木桥”

3. 锁升级过程

问题： synchronized锁升级的过程是什么？

详细答案：

  1. 无锁

    • 初始状态
    • 没有线程访问

  2. 偏向锁

    • 只有一个线程访问
    • 记录线程ID

  3. 轻量级锁

    • 多个线程交替访问
    • 使用CAS操作

  4. 重量级锁

    • 多个线程同时访问
    • 使用操作系统互斥量

生活化比喻：

  • 就像”图书馆座位系统”
    • 无锁：座位空闲
    • 偏向锁：常客预约
    • 轻量级锁：临时使用
    • 重量级锁：高峰期管理

五、实战技巧

1. 锁的选择

选择原则：

  1. 优先使用synchronized

    • 代码简单
    • 性能足够
    • 自动释放

  2. 需要特殊功能时使用ReentrantLock

    • 需要超时机制
    • 需要响应中断
    • 需要公平锁

生活化比喻：

  • 就像选择交通工具
    • synchronized像”公交车”，简单方便
    • ReentrantLock像”出租车”，更灵活

2. 锁的使用建议

使用建议：

  1. 尽量缩小锁的范围
  2. 避免嵌套锁
  3. 使用try-finally确保释放锁
  4. 注意锁的性能开销

代码示例：

public class LockBestPractice {
    private final ReentrantLock lock = new ReentrantLock();
    
    public void goodPractice() {
        // 1. 缩小锁范围
        {
            lock.lock();
            try {
                // 最小范围的同步代码
            } finally {
                lock.unlock();
            }
        }
        
        // 2. 避免嵌套锁
        // 不好的做法
        synchronized(lock1) {
            synchronized(lock2) {
                // 可能导致死锁
            }
        }
    }
}

生活化比喻：

  • 就像”排队买票”
    • 缩小范围：只锁住收银台
    • 避免嵌套：不要同时排多个队
    • 确保释放：买完票就离开

3. 性能优化技巧

优化方法：

  1. 使用读写锁
  2. 使用分段锁
  3. 使用原子类
  4. 使用无锁算法

代码示例：

public class PerformanceOptimization {
    // 1. 使用读写锁
    private final ReadWriteLock rwLock = new ReentrantReadWriteLock();
    
    // 2. 使用分段锁
    private final Segment[] segments = new Segment[16];
    
    // 3. 使用原子类
    private final AtomicInteger counter = new AtomicInteger(0);
    
    // 4. 使用无锁算法
    public void lockFreeAlgorithm() {
        while(true) {
            int current = counter.get();
            if(counter.compareAndSet(current, current + 1)) {
                break;
            }
        }
    }
}

生活化比喻：

  • 就像”图书馆管理”
    • 读写锁：读者和管理员分开
    • 分段锁：不同区域分开管理
    • 原子类：自动计数
    • 无锁算法：自助借还书
