深入理解Java之多线程与IO学习笔记

第一部分：Java多线程基础

什么是线程

启动一个java程序实际上是启动了一个JVM进程，然后JVM创建主线程让其执行main方法，在main方法中我们又可以创建启动其他子线程。

  • 进程是资源分配的最小单位-比如正在运行的百度网盘
  • 线程是程序执行的最小单位，线程属于进程。比如百度网盘下载

线程的生命周期

线程的生命周期就像人的一生，有出生、成长、工作、休息和结束等阶段：

  1. 新建（New）：线程被创建，但还没有调用start()方法
  2. 就绪（Runnable）：线程已经启动，等待CPU分配时间片
  3. 运行（Running）：线程正在执行
  4. 阻塞（Blocked）：线程因为某些原因暂时停止执行
  5. 等待（Waiting）：线程等待其他线程的通知
  6. 超时等待（Timed Waiting）：线程等待一段时间后自动恢复
  7. 终止（Terminated）：线程执行完毕

线程的优先级

线程的优先级就像餐厅服务员的等级，优先级高的线程更容易获得CPU的时间片。优先级范围是1-10，默认是5。

public class ThreadPriority {
    public static void main(String[] args) {
        Thread thread1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("高优先级线程: " + i);
            }
        });
        thread1.setPriority(Thread.MAX_PRIORITY); // 设置最高优先级
        
        Thread thread2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("低优先级线程: " + i);
            }
        });
        thread2.setPriority(Thread.MIN_PRIORITY); // 设置最低优先级
        
        thread1.start();
        thread2.start();
    }
}

守护线程

守护线程就像餐厅的清洁工，当所有顾客（非守护线程）都离开后，清洁工也会自动离开。守护线程会随着主线程的结束而结束。

public class DaemonThread {
    public static void main(String[] args) {
        Thread daemonThread = new Thread(() -> {
            while (true) {
                System.out.println("守护线程正在运行...");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });
        daemonThread.setDaemon(true); // 设置为守护线程
        daemonThread.start();
        
        // 主线程休眠5秒后结束
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println("主线程结束，守护线程也会结束");
    }
}

第二部分：线程同步与锁机制

线程安全问题

想象一个银行账户，如果多个线程同时对这个账户进行存取款操作，可能会导致数据不一致。这就是线程安全问题。

public class BankAccount {
    private int balance = 1000;
    
    // 不安全的取款方法
    public void withdraw(int amount) {
        if (balance >= amount) {
            try {
                Thread.sleep(100); // 模拟处理时间
                balance -= amount;
                System.out.println("取款成功，余额：" + balance);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
    
    public static void main(String[] args) {
        BankAccount account = new BankAccount();
        
        // 创建多个线程同时取款
        for (int i = 0; i < 5; i++) {
            new Thread(() -> {
                account.withdraw(100);
            }).start();
        }
    }
}

synchronized关键字

synchronized就像银行柜台的锁，当一个客户在办理业务时，其他客户必须等待。

public class SafeBankAccount {
    private int balance = 1000;
    
    // 使用synchronized关键字保护方法
    public synchronized void withdraw(int amount) {
        if (balance >= amount) {
            try {
                Thread.sleep(100);
                balance -= amount;
                System.out.println("取款成功，余额：" + balance);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
    
    // 使用synchronized代码块
    public void deposit(int amount) {
        synchronized (this) {
            try {
                Thread.sleep(100);
                balance += amount;
                System.out.println("存款成功，余额：" + balance);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

volatile关键字

volatile就像餐厅的公告板，当一个服务员更新了公告板的内容，其他服务员都能立即看到最新的信息。

public class VolatileExample {

    public void setFlag() {
        flag = true;
    }
    
    public void doWork() {

原子类

原子类就像自动售货机，它的操作是不可分割的，要么全部成功，要么全部失败。

Lock接口

Lock接口提供了比synchronized更灵活的锁机制，就像银行的自助取款机，可以设置超时时间。

import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class LockExample {
    private Lock lock = new ReentrantLock();
    private int balance = 1000;
    
    public void withdraw(int amount) {
        try {
            // 尝试获取锁，最多等待1秒
            if (lock.tryLock(1, TimeUnit.SECONDS)) {
                try {
                    if (balance >= amount) {
                        Thread.sleep(100);
                        balance -= amount;
                        System.out.println("取款成功，余额：" + balance);
                    }
                } finally {
                    lock.unlock(); // 确保释放锁
                }
            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

第三部分：线程池

线程池的概念

线程池就像餐厅的固定员工团队，不需要每次都招聘新员工，而是复用现有的员工。

线程池的工作原理

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolExample {
    public static void main(String[] args) {
        // 创建固定大小的线程池
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        // 提交任务
        for (int i = 0; i < 5; i++) {
            final int taskId = i;
            executor.submit(() -> {
                System.out.println("任务" + taskId + "正在执行");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            });
        }
        
        // 关闭线程池
        executor.shutdown();
    }
}

线程池的创建方式

import java.util.concurrent.*;

public class ThreadPoolCreation {
    public static void main(String[] args) {
        // 1. 固定大小的线程池
        ExecutorService fixedPool = Executors.newFixedThreadPool(3);

        ExecutorService singlePool = Executors.newSingleThreadExecutor();
        
        // 3. 可缓存的线程池
        ExecutorService cachedPool = Executors.newCachedThreadPool();
        
        // 4. 定时执行的线程池
        ScheduledExecutorService scheduledPool = Executors.newScheduledThreadPool(3);
        
        // 5. 自定义线程池
        ThreadPoolExecutor customPool = new ThreadPoolExecutor(
            2,                      // 核心线程数
            5,                      // 最大线程数
            60L,                    // 空闲线程存活时间
            TimeUnit.SECONDS,       // 时间单位
            new LinkedBlockingQueue<>(10),  // 工作队列
            new ThreadFactory() {   // 线程工厂

                }

            new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
        );
    }
}

第四部分：并发工具类

CountDownLatch

CountDownLatch就像运动会的起跑线，所有运动员都准备好后，才能开始比赛。

import java.util.concurrent.CountDownLatch;

public class CountDownLatchExample {
    public static void main(String[] args) {
        CountDownLatch latch = new CountDownLatch(3);
        
        // 创建三个线程
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                try {
                    System.out.println("线程" + Thread.currentThread().getName() + "准备就绪");
                    Thread.sleep(1000);
                    latch.countDown(); // 计数减1
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }).start();
        }
        
        try {
            latch.await(); // 等待所有线程准备就绪
            System.out.println("所有线程准备就绪，开始执行！");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }
}

CyclicBarrier

CyclicBarrier就像朋友约好一起出发，所有人都到达集合地点后，才能一起出发。

import java.util.concurrent.CyclicBarrier;

public class CyclicBarrierExample {
    public static void main(String[] args) {
        CyclicBarrier barrier = new CyclicBarrier(3, () -> {
            System.out.println("所有线程都到达集合点，开始下一轮！");
        });
        
        // 创建三个线程
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                try {
                    System.out.println("线程" + Thread.currentThread().getName() + "到达集合点");
                    barrier.await(); // 等待其他线程
                } catch (Exception e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}

Semaphore

Semaphore就像停车场的车位，只有有限数量的车位，当车位满了，其他车需要等待。

import java.util.concurrent.Semaphore;

public class SemaphoreExample {
    public static void main(String[] args) {
        Semaphore semaphore = new Semaphore(3); // 3个车位
        
        // 创建5辆车
        for (int i = 0; i < 5; i++) {
            new Thread(() -> {
                try {
                    semaphore.acquire(); // 获取车位
                    System.out.println("车辆" + Thread.currentThread().getName() + "停车");
                    Thread.sleep(2000); // 停车2秒
                    semaphore.release(); // 释放车位
                    System.out.println("车辆" + Thread.currentThread().getName() + "离开");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }).start();
        }
    }
}

第六部分：NIO

NIO的概念

NIO（New IO）就像高速公路，可以双向通行，而且可以同时处理多个通道。

Buffer

import java.nio.ByteBuffer;

public class BufferExample {
    public static void main(String[] args) {
        // 创建缓冲区
        ByteBuffer buffer = ByteBuffer.allocate(1024);
        
        // 写入数据
        buffer.put("Hello, World!".getBytes());
        
        // 切换到读模式
        buffer.flip();
        
        // 读取数据
        while (buffer.hasRemaining()) {
            System.out.print((char) buffer.get());
        }
        
        // 清空缓冲区
        buffer.clear();
    }
}

Selector

import java.net.InetSocketAddress;
import java.nio.ByteBuffer;
import java.nio.channels.*;
import java.util.Iterator;
import java.util.Set;

public class SelectorExample {
    public static void main(String[] args) {
        try {
            // 创建选择器
            Selector selector = Selector.open();
            
            // 创建服务器通道
            ServerSocketChannel serverChannel = ServerSocketChannel.open();
            serverChannel.configureBlocking(false);
            serverChannel.bind(new InetSocketAddress(8080));
            serverChannel.register(selector, SelectionKey.OP_ACCEPT);
            
            while (true) {
                // 选择准备好的通道
                selector.select();
                
                // 获取选择的键集合
                Set<SelectionKey> selectedKeys = selector.selectedKeys();
                Iterator<SelectionKey> iter = selectedKeys.iterator();
                
                while (iter.hasNext()) {
                    SelectionKey key = iter.next();
                    
                    if (key.isAcceptable()) {
                        // 处理接受连接
                        ServerSocketChannel server = (ServerSocketChannel) key.channel();
                        SocketChannel client = server.accept();
                        client.configureBlocking(false);
                        client.register(selector, SelectionKey.OP_READ);
                    }
                    
                    if (key.isReadable()) {
                        // 处理读取数据
                        SocketChannel client = (SocketChannel) key.channel();
                        ByteBuffer buffer = ByteBuffer.allocate(1024);
                        client.read(buffer);
                        buffer.flip();
                        client.write(buffer);
                    }
                    
                    iter.remove();
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Reactor模式

import java.net.InetSocketAddress;
import java.nio.channels.*;
import java.util.Iterator;
import java.util.Set;

public class ReactorServer {
    public static void main(String[] args) {
        try {
            // 创建选择器
            Selector selector = Selector.open();
            
            // 创建服务器通道
            ServerSocketChannel serverChannel = ServerSocketChannel.open();
            serverChannel.configureBlocking(false);
            serverChannel.bind(new InetSocketAddress(8080));
            serverChannel.register(selector, SelectionKey.OP_ACCEPT);
            
            // 创建线程池
            ExecutorService executor = Executors.newFixedThreadPool(4);
            
            while (true) {
                selector.select();
                Set<SelectionKey> selectedKeys = selector.selectedKeys();
                Iterator<SelectionKey> iter = selectedKeys.iterator();
                
                while (iter.hasNext()) {
                    SelectionKey key = iter.next();
                    
                    if (key.isAcceptable()) {
                        // 处理接受连接
                        ServerSocketChannel server = (ServerSocketChannel) key.channel();
                        SocketChannel client = server.accept();
                        client.configureBlocking(false);
                        client.register(selector, SelectionKey.OP_READ);
                    }
                    
                    if (key.isReadable()) {
                        // 将读取操作提交到线程池
                        executor.submit(() -> {
                            try {
                                SocketChannel client = (SocketChannel) key.channel();
                                ByteBuffer buffer = ByteBuffer.allocate(1024);
                                client.read(buffer);
                                buffer.flip();
                                client.write(buffer);
                            } catch (IOException e) {
                                e.printStackTrace();
                            }
                        });
                    }
                    
                    iter.remove();
                }
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Proactor模式

import java.net.InetSocketAddress;
import java.nio.channels.*;
import java.util.concurrent.*;

public class ProactorServer {
    public static void main(String[] args) {
        try {
            // 创建异步服务器通道
            AsynchronousServerSocketChannel serverChannel = 
                AsynchronousServerSocketChannel.open();
            serverChannel.bind(new InetSocketAddress(8080));
            
            // 创建线程池
            ExecutorService executor = Executors.newFixedThreadPool(4);
            
            // 接受连接
            serverChannel.accept(null, new CompletionHandler<AsynchronousSocketChannel, Void>() {
                @Override
                public void completed(AsynchronousSocketChannel client, Void attachment) {
                    // 继续接受下一个连接
                    serverChannel.accept(null, this);
                    
                    // 处理当前连接
                    handleClient(client);
                }
                
                @Override
                public void failed(Throwable exc, Void attachment) {
                    exc.printStackTrace();
                }
            });
            
            // 保持服务器运行
            Thread.sleep(Integer.MAX_VALUE);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    
    private static void handleClient(AsynchronousSocketChannel client) {
        ByteBuffer buffer = ByteBuffer.allocate(1024);
        
        // 读取数据
        client.read(buffer, buffer, new CompletionHandler<Integer, ByteBuffer>() {
            @Override
            public void completed(Integer result, ByteBuffer attachment) {
                attachment.flip();
                // 写入响应
                client.write(attachment, attachment, new CompletionHandler<Integer, ByteBuffer>() {
                    @Override
                    public void completed(Integer result, ByteBuffer attachment) {
                        try {
                            client.close();
                        } catch (IOException e) {
                            e.printStackTrace();
                        }
                    }
                    
                    @Override
                    public void failed(Throwable exc, ByteBuffer attachment) {
                        exc.printStackTrace();
                    }
                });
            }
            
            @Override
            public void failed(Throwable exc, ByteBuffer attachment) {
                exc.printStackTrace();
            }
        });
    }
}

最佳实践

  1. 选择合适的IO模型

    • 对于简单的应用，使用传统的阻塞IO
    • 对于高并发应用，使用NIO或AIO
    • 对于需要大量连接的应用，使用Reactor或Proactor模式

  2. 合理使用线程池

    • 根据应用特点选择合适的线程池类型
    • 设置合适的线程池参数
    • 使用线程池监控工具

  3. 注意资源管理

    • 使用try-with-resources语句
    • 及时关闭不需要的资源
    • 避免资源泄漏

  4. 异常处理

    • 正确处理IO异常
    • 实现优雅的关闭机制
    • 记录关键日志

  5. 性能优化

    • 使用缓冲流提高IO效率
    • 合理设置缓冲区大小
    • 避免频繁的IO操作
