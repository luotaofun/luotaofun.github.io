深入理解Java之多线程与IO学习笔记

第一部分：Java多线程基础

什么是线程

启动一个java程序实际上是启动了一个JVM进程，然后JVM创建主线程让其执行main方法，在main方法中我们又可以创建启动其他子线程。

  • 进程是资源分配的最小单位-比如正在运行的百度网盘
  • 线程是程序执行的最小单位，线程属于进程。比如百度网盘下载

线程的生命周期

线程的生命周期就像人的一生，有出生、成长、工作、休息和结束等阶段：

  1. 新建（New）：线程被创建，但还没有调用start()方法
  2. 就绪（Runnable）：线程已经启动，等待CPU分配时间片
  3. 运行（Running）：线程正在执行
  4. 阻塞（Blocked）：线程因为某些原因暂时停止执行
  5. 等待（Waiting）：线程等待其他线程的通知
  6. 超时等待（Timed Waiting）：线程等待一段时间后自动恢复
  7. 终止（Terminated）：线程执行完毕

线程的优先级

Java中的线程调度方式是抢占式调度模型，系，优先级高的线程更容易获得CPU的时间片，若优先级相同则会随机选择一个。默认优先级值NORM_PRIORITY=5

public class ThreadPriority {
    public static void main(String[] args) {
        Thread thread1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("高优先级线程: " + i);
            }
        });
        thread1.setPriority(Thread.MAX_PRIORITY); // 设置最高优先级10
        
        Thread thread2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println("低优先级线程: " + i);
            }
        });
        thread2.setPriority(Thread.MIN_PRIORITY); // 设置最低优先级1
        
        thread1.start();
        thread2.start();
    }
}

守护线程

守护线程就像餐厅的清洁工，当所有顾客（非守护线程）都离开后，清洁工也会自动离开。守护线程会随着主线程的结束而结束。

当运行的线程都是守护线程时，JVM结束运行。

public class DaemonThread {
    public static void main(String[] args) {
        Thread daemonThread = new Thread(() -> {
            while (true) {
                System.out.println("守护线程正在运行...");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        });
        daemonThread.setDaemon(true); // 设置为守护线程
        daemonThread.start();
        
        // 主线程休眠5秒后结束
        try {
            Thread.sleep(5000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        System.out.println("主线程结束，守护线程也会结束");
    }
}

线程安全问题

想象一个银行账户，如果多个线程同时对这个账户进行存取款操作，可能会导致数据不一致。这就是线程安全问题。

public class BankAccount {

            try {

            }

    }

}

synchronized关键字

synchronized就像银行柜台的锁，当一个客户在办理业务时，其他客户必须等待。

public class SafeBankAccount {
    private int balance = 1000;
    
    // 使用synchronized关键字保护方法
    public synchronized void withdraw(int amount) {

            try {
                Thread.sleep(100);

                e.printStackTrace();
            }
        }

    // 使用synchronized代码块

            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

Lock接口

Lock接口提供了比synchronized更灵活的锁机制，就像银行的自助取款机，可以设置超时时间。

想象一个房间门上的锁，只有持有钥匙（锁对象）的人才能进入，并且如果同一个人再次进入，不需要重新获取钥匙。

锁的定义：ReentrantLock 需要显式创建锁对象，而 synchronized 是隐式的。
锁的获取与释放：ReentrantLock 需要手动调用 lock() 和 unlock()，而 synchronized 自动管理。
功能扩展：ReentrantLock 提供更多功能，如尝试加锁（tryLock()）、设置超时等。

package com.luotao.job.utils.demo;

import java.util.ArrayList;
import java.util.Collections;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

// 摇号系统
class CarLottery implements Runnable {
    private ArrayList<String> serialNumber;// 申请编码池
    private int count;// 本期指标数
    private final Lock lock = new ReentrantLock();

    public CarLottery(ArrayList<String> serialNumber, int count) {
        this.serialNumber = serialNumber;
        this.count = count;
    }

    @Override
    public void run() {
        while (true) {
            // 尝试获取锁，最多等待1秒
            try {
                if (lock.tryLock(1, TimeUnit.SECONDS)) {
                    try {
                        if (count == 0) {
                            break;
                        } else {
                            Collections.shuffle(serialNumber);// 随机打乱元素顺序
                            String selectNumber = serialNumber.remove(0);
                            System.out.println(Thread.currentThread().getName() + "抽中" + selectNumber);
                            count--;
                            try {
                                Thread.sleep(200);
                            } catch (InterruptedException e) {
                                throw new RuntimeException(e);
                            }
                        }
                    } finally {
                        // 确保锁被释放
                        lock.unlock();
                    }
                }
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }
        }
    }
}


class App {
    public static void main(String[] args) {
        ArrayList<String> serialNumber = new ArrayList<>();
        // 将编码批量放入集合中
        Collections.addAll(serialNumber,
                "A001", "A002", "A003", "A004", "A005", "A006", "A007", "A008", "A009", "A010");
        CarLottery carLottery = new CarLottery(serialNumber, 5);
        Thread thread1 = new Thread(carLottery);
        Thread thread2 = new Thread(carLottery);

    }
}

死锁

package com.luotao.job.utils.demo;

import java.util.concurrent.TimeUnit;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

class DeadLockDemo {
    Object lock1 = new Object();
    Object lock2 = new Object();

    // 从左侧进入库房的线程类
    public class ThreadLeft extends Thread {
        @Override
        public void run() {
            synchronized (lock1) {
                System.out.println("线程Left获得了lock1");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("线程Left尝试获取lock2");
                synchronized (lock2) {
                    System.out.println("线程Left获得了lock2");
                    try {
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
            }
            System.out.println("线程Left执行完毕");
        }

    }

    // 从右侧侧进入库房的线程类
    public class ThreadRight extends Thread {
        @Override
        public void run() {
            synchronized (lock2) {
                System.out.println("线程Right获得了lock2");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("线程Right尝试获取lock1");
                synchronized (lock1) {
                    System.out.println("线程Right获得了lock1");
                    try {
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println("线程Right执行完毕");
            }

        }
    }
    // 从右侧侧进入库房的线程类:统一锁获取顺序避免死锁
    public class ThreadRight2 extends Thread {
        @Override
        public void run() {
            synchronized (lock1) {
                System.out.println("线程Right获得了lock1");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                System.out.println("线程Right尝试获取lock2");
                synchronized (lock2) {
                    System.out.println("线程Right获得了lock2");
                    try {
                        Thread.sleep(1000);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                }
                System.out.println("线程Right执行完毕");
            }

        }
    }
    // 从右侧侧进入库房的线程类:使用ReentrantLock避免死锁
    public class ThreadRightLock extends Thread {
        @Override
        public void run() {
            Lock lockA = new ReentrantLock();
            Lock lockB = new ReentrantLock();
            boolean getLockA = false;
            boolean getLockB = false;

            try {
                // 尝试获取第一把锁，最多等待2秒
                getLockA = lockB.tryLock(2, TimeUnit.SECONDS);
                if (getLockA) {
                    System.out.println("线程RightLock获得了lockB");

                    // 尝试获取第二把锁，最多等待2秒
                    getLockB = lockA.tryLock(2, TimeUnit.SECONDS);
                    if (getLockB) {
                        System.out.println("线程RightLock获得了lockA");
                        // 使用两把锁执行业务逻辑
                        System.out.println("线程RightLock成功获取两把锁，执行业务逻辑");
                        try {
                            Thread.sleep(1000);
                        } catch (InterruptedException e) {
                            e.printStackTrace();
                        }
                    } else {
                        // 获取第二把锁失败，放弃任务
                        System.out.println("线程RightLock获取lockA超时，放弃任务");
                    }
                } else {
                    // 获取第一把锁失败，放弃任务
                    System.out.println("线程RightLock获取lockB超时，放弃任务");
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
            } finally {
                // 释放已获得的锁
                if (getLockB) {
                    lockA.unlock();
                    System.out.println("线程RightLock释放了lockA");
                }
                if (getLockA) {
                    lockB.unlock();
                    System.out.println("线程RightLock释放了lockB");
                }
            }
            System.out.println("线程RightLock执行完毕");
        }
    }
}

避免死锁的方法

死锁是指两个或多个线程互相等待对方持有的锁，导致这些线程永远处于阻塞状态。避免死锁的方法包括：按固定顺序获取锁、使用tryLock()方法、设置锁超时、使用线程池和并发工具类等。

死锁的四个必要条件是：互斥条件、请求与保持条件、不剥夺条件和循环等待条件。只要破坏其中一个条件，就能避免死锁。

1. 统一锁获取顺序

相同的锁获取顺序：先lock1，再lock2。这样可以避免循环等待条件，从而避免死锁。

2. 使用ReentrantLock的tryLock方法
synchronized和ReentrantLock在处理死锁方面有什么区别？

ReentrantLock提供了更多避免死锁的机制，如tryLock()方法（支持超时）、可中断的锁获取、公平锁选项等，而synchronized只提供了基本的互斥功能。

第三部分：线程池

线程池的概念

线程池就像餐厅的固定员工团队，不需要每次都招聘新员工，而是复用现有的员工。

线程池的工作原理

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolExample {
    public static void main(String[] args) {
        // 创建固定大小的线程池
        ExecutorService executor = Executors.newFixedThreadPool(3);
        
        // 提交任务
        for (int i = 0; i < 5; i++) {
            final int taskId = i;
            executor.submit(() -> {
                System.out.println("任务" + taskId + "正在执行");
                try {
                    Thread.sleep(1000);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            });
        }
        
        // 关闭线程池
        executor.shutdown();
    }
}

                }

            new ThreadPoolExecutor.CallerRunsPolicy()  // 拒绝策略
        );
    }
}

第四部分：并发工具类

CountDownLatch

CountDownLatch就像运动会的起跑线，所有运动员都准备好后，才能开始比赛。

import java.util.concurrent.CountDownLatch;

public class CountDownLatchExample {
    public static void main(String[] args) {
        CountDownLatch latch = new CountDownLatch(3);
        
        // 创建三个线程
        for (int i = 0; i < 3; i++) {
            new Thread(() -> {
                try {
                    System.out.println("线程" + Thread.currentThread().getName() + "准备就绪");
                    Thread.sleep(1000);
                    latch.countDown(); // 计数减1
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }).start();
        }
        
        try {
            latch.await(); // 等待所有线程准备就绪
            System.out.println("所有线程准备就绪，开始执行！");
        } catch (InterruptedException e) {

IO 基础概念

Java IO 的核心概念

  1. 流（Stream）：是一个抽象的概念，代表了数据的流动。就像水流一样，数据从一个地方流到另一个地方。

  2. 字节（Byte）与字符（Character）：

    • 字节是计算机存储的基本单位（1字节=8位）
    • 字符是人类可读的符号（在Java中，一个字符通常占用2个字节，使用UTF-16编码）

  3. 阻塞与非阻塞：

    • 传统IO是阻塞式的，就像排队取水，一个人取完才能轮到下一个
    • NIO提供了非阻塞式IO，就像多个水龙头同时供水

字节流与字符流

Java IO 流主要分为两大类：字节流和字符流。

字节流

字节流适合处理二进制文件（如图片、音频、视频等非文本文件）。

基本类：

  • InputStream：所有字节输入流的抽象基类
  • OutputStream：所有字节输出流的抽象基类

常用实现类：

  • FileInputStream/FileOutputStream：文件字节流
  • ByteArrayInputStream/ByteArrayOutputStream：字节数组流
  • DataInputStream/DataOutputStream：处理基本数据类型
  • ObjectInputStream/ObjectOutputStream：对象序列化流

字符流

字符流处理字符数据，适合处理文本文件，如配置文件。字符流内部使用字节流，但会自动进行字符编码和解码。

字符输入流（Reader抽象类）

方法名 返回值 描述
read() 读完返回-1 从输入流读一个字符
read(char[] cbuf) 读完返回-1 读字符到字符数组中
close() void 关闭输入流，释放系统资源

常用实现类：

  • FileReader/FileWriter：文件字符流
  • InputStreamReader/OutputStreamWriter：字节流转字符流的桥梁
  • BufferedReader/BufferedWriter：带缓冲的字符流
  • StringReader/StringWriter：字符串流
字符输入流（FileReader和BufferedReader）
  • 创建FileReader对象——>read()读——>-1——>关闭流对象（判空）
  • 创建BufferedReader对象——>readLine——>null——>关闭流对象（判空）
package com.luotao.job.utils.demo;

import java.io.*;
import java.util.ArrayList;
import java.util.Iterator;

class DataRead {
    private String fileName;
    private FileReader fileReader;
    private BufferedReader bufferedReader;

    public DataRead(String fileName) {
        this.fileName = fileName;
    }

    public DataRead(FileReader fileReader) {
        this.fileReader = fileReader;
    }

    public DataRead(BufferedReader bufferedReader) {
        this.bufferedReader = bufferedReader;
    }


    /**
     * @Description: 字符输入流
     **/
    public ArrayList<String> read() {
        ArrayList<String> arrayList = new ArrayList<>();// 保存读到的每行

        FileReader fileReader = null;
        BufferedReader bufferedReader=null;
        try {
            fileReader = new FileReader(fileName); // 创建输入流对象
            while (true) {
                // 输入流对象按单个字符读取，返回-1表示读取完毕
                int ch = fileReader.read();
                if (ch == -1) {
                    break;
                }
                System.out.println((char) ch);
            }

        } catch (FileNotFoundException e) {
            System.out.println(e.getMessage());
        } catch (IOException e) {
            System.out.println(e.getMessage());
        } finally {
            try {
                if (fileReader != null ) {
                    // 先要判空，不然会空指针
                    fileReader.close();
                }
            } catch (IOException e) {
                throw new RuntimeException(e);
            }
        }
        return arrayList;
    }

    /**
     * @Description: 字符输入流
     **/
    public ArrayList<String> readByBuf() {
        ArrayList<String> arrayList = new ArrayList<>();// 保存读到的每行

        FileReader fileReader = null;
        BufferedReader bufferedReader=null;
        try {
            fileReader = new FileReader(fileName); // 创建输入流对象
            bufferedReader = new BufferedReader(fileReader);// 通过输入流对象创建缓冲流对象
            while (true) {
                // 缓冲流对象按行读取,返回null表示读取完毕
                String str = bufferedReader.readLine();
                if (str == null) {
                    break;
                }
                if (!str.isEmpty()){
                    arrayList.add(str.trim());
                }
            }
        } catch (FileNotFoundException e) {
            System.out.println(e.getMessage());
        } catch (IOException e) {
            System.out.println(e.getMessage());
        } finally {
            try {
                if (fileReader != null && bufferedReader != null) {
                    // 先要判空，不然会空指针
                    fileReader.close();
                    bufferedReader.close();
                }
            } catch (IOException e) {
                throw new RuntimeException(e);
            }
        }
        return arrayList;
    }
}

对象序列化

序列化是将对象转换为字节序列的过程，反序列化则是将字节序列恢复为对象的过程。就像把一个立体物品压扁后通过邮件发送，收件人再将其还原成原来的形状。

实现序列化

  1. 类必须实现 java.io.Serializable 接口
  2. 使用 ObjectOutputStream 写入对象
  3. 使用 ObjectInputStream 读取对象
// 定义可序列化的类
class Person implements Serializable {
    private static final long serialVersionUID = 1L; // 序列化版本号
    private String name;
    private int age;
    
    // 构造方法、getter和setter省略
}

// 序列化对象
try (ObjectOutputStream oos = new ObjectOutputStream(
        new FileOutputStream("person.dat"))) {
    Person person = new Person("张三", 25);
    oos.writeObject(person);
} catch (IOException e) {
    e.printStackTrace();
}

// 反序列化对象
try (ObjectInputStream ois = new ObjectInputStream(
        new FileInputStream("person.dat"))) {
    Person person = (Person) ois.readObject();
    System.out.println(person.getName() + ", " + person.getAge());
} catch (IOException | ClassNotFoundException e) {
    e.printStackTrace();
}

注意事项

  1. transient 关键字：标记不需要序列化的字段
  2. static 字段不会被序列化
  3. 序列化版本号 serialVersionUID 用于版本控制

NIO 基础

NIO (New IO) 是 Java 1.4 引入的 IO API，提供了比传统 IO 更高效的实现。

NIO 的核心组件

  1. Buffer（缓冲区）：数据的临时存储区域
  2. Channel（通道）：数据传输的通道，类似于流但可以双向传输
  3. Selector（选择器）：用于监控多个通道的IO状态

基本用法示例

// 使用NIO读取文件
try {
    RandomAccessFile file = new RandomAccessFile("nio_test.txt", "r");
    FileChannel channel = file.getChannel();
    
    // 创建缓冲区
    ByteBuffer buffer = ByteBuffer.allocate(1024);
    
    // 读取数据到缓冲区
    int bytesRead = channel.read(buffer);
    while (bytesRead != -1) {
        // 切换到读模式
        buffer.flip();
        
        // 读取缓冲区数据
        while (buffer.hasRemaining()) {
            System.out.print((char) buffer.get());
        }
        
        // 清空缓冲区，准备下一次读取
        buffer.clear();
        bytesRead = channel.read(buffer);
    }
    
    file.close();
} catch (IOException e) {
    e.printStackTrace();
}

常见面试题

1. Java IO 与 NIO 的区别？

特性 IO NIO
数据处理方式 面向流 面向缓冲区
处理方式 阻塞IO 支持非阻塞IO
选择器 无 有（可监控多个通道）
适用场景 连接数少，数据量大 连接数多，数据量少

2. 字节流和字符流的区别？

  • 字节流直接操作字节，适合处理二进制文件
  • 字符流操作字符，内部进行编码转换，适合处理文本文件
  • 字符流 = 字节流 + 编码/解码

3. 什么是序列化和反序列化？

  • 序列化：将对象转换为字节序列的过程
  • 反序列化：将字节序列恢复为对象的过程
  • 用途：对象持久化、网络传输

4. 如何提高IO性能？

  • 使用缓冲流
  • 使用NIO的内存映射文件
  • 使用更大的缓冲区
  • 减少IO操作次数
  • 使用并行流处理

5. transient关键字的作用？

  • 标记不需要被序列化的字段
  • 通常用于敏感信息（如密码）或可以由其他字段计算得出的冗余字段
