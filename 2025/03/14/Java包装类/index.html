Java包装类

发表于 2025-03-14

Java包装类详解

包装类基础概念

什么是包装类？

包装类是将Java的基本数据类型（如int、char等）封装成对象的类。想象一下，基本类型就像是一颗裸露的钻石，而包装类就是给这颗钻石精心设计的包装盒，不仅保护了钻石，还赋予了它更多的功能。

包装类的主要作用：

  1. 将基本数据类型转换为对象，使其能在对象世界中”生存”
  2. 提供很多实用的方法，如类型转换、进制转换等
  3. 支持null值，而基本类型不支持
  4. 在集合类中使用（因为集合类只能存储对象）

为什么需要包装类？

在Java中，有两个世界：基本类型的世界和对象的世界。有时候，这两个世界需要沟通，包装类就是这座桥梁。

例如：

  • Java集合（如ArrayList、HashMap等）只能存储对象，不能直接存储基本类型
  • 泛型不支持基本类型，只支持对象类型
  • 需要在基本类型上调用方法时
  • 需要表示基本类型的null值时

基本类型与包装类对应关系

Java中的每种基本类型都有对应的包装类：

基本类型 包装类 父类
byte Byte Number
short Short Number
int Integer Number
long Long Number
float Float Number
double Double Number
char Character Object
boolean Boolean Object

所有的数值型包装类（Byte、Short、Integer、Long、Float、Double）都继承自抽象类Number，而Character和Boolean直接继承自Object。

装箱与拆箱

装箱（Boxing）

装箱是将基本类型转换为对应的包装类对象。就像把钻石放入精美的包装盒中。

手动装箱：

int primitiveInt = 42;
Integer wrappedInt = Integer.valueOf(primitiveInt); // 手动装箱

自动装箱（Java 5及以上）：

int primitiveInt = 42;
Integer wrappedInt = primitiveInt; // 自动装箱

拆箱（Unboxing）

拆箱是将包装类对象转换为对应的基本类型。就像从包装盒中取出钻石。

手动拆箱：

Integer wrappedInt = Integer.valueOf(42);
int primitiveInt = wrappedInt.intValue(); // 手动拆箱

自动拆箱（Java 5及以上）：

Integer wrappedInt = 42;
int primitiveInt = wrappedInt; // 自动拆箱

自动装箱/拆箱的实例

public class AutoBoxingUnboxingDemo {
    public static void main(String[] args) {
        // 自动装箱
        Integer num1 = 100;
        
        // 自动拆箱
        int num2 = num1;
        
        // 在表达式中的自动装箱和拆箱
        Integer num3 = 100;
        Integer num4 = 200;
        int result = num3 + num4; // num3和num4自动拆箱，计算后结果为基本类型
        
        // 在方法调用中
        printNumber(100); // 自动装箱
    }
    
    public static void printNumber(Integer num) {
        System.out.println(num);
    }
}

自动装箱/拆箱的原理

自动装箱/拆箱是Java编译器提供的语法糖，在编译时会自动转换为对应的方法调用：

  • 自动装箱：Integer.valueOf(int)
  • 自动拆箱：intValue()

例如，以下代码：

Integer num = 100;
int value = num;

编译后等价于：

Integer num = Integer.valueOf(100);
int value = num.intValue();

包装类的缓存机制

为了提高性能，Java的包装类对常用的值实现了缓存机制。这就像商店里常用商品会放在触手可及的地方，而不常用的商品则需要去仓库取。

Integer缓存

Integer类的缓存范围是-128到127（可通过JVM参数调整上限）。在这个范围内的整数会被缓存，超出范围则创建新对象。

public class IntegerCacheDemo {
    public static void main(String[] args) {
        Integer num1 = 127;
        Integer num2 = 127;
        System.out.println(num1 == num2); // true，使用了缓存的同一个对象
        
        Integer num3 = 128;
        Integer num4 = 128;
        System.out.println(num3 == num4); // false，超出缓存范围，创建了不同对象
        
        // valueOf方法使用缓存
        Integer num5 = Integer.valueOf(127);
        Integer num6 = Integer.valueOf(127);
        System.out.println(num5 == num6); // true
        
        // 构造方法不使用缓存（已废弃）
        Integer num7 = new Integer(127);
        Integer num8 = new Integer(127);
        System.out.println(num7 == num8); // false，每次都创建新对象
    }
}

其他包装类的缓存

  • Boolean: 缓存了true和false对象
  • Byte: 缓存了所有可能的值（-128到127）
  • Short: 缓存了-128到127
  • Character: 缓存了0到127
  • Long: 缓存了-128到127
  • Float和Double: 没有实现缓存机制

包装类的比较

==与equals的区别

  • ==: 比较引用是否指向同一对象（对于包装类）或值是否相等（对于基本类型）
  • equals(): 比较包装类对象的值是否相等
public class WrapperComparisonDemo {
    public static void main(String[] args) {
        Integer num1 = 100;
        Integer num2 = 100;
        Integer num3 = 200;
        Integer num4 = 200;
        
        // 使用==比较
        System.out.println(num1 == num2);  // true，因为100在缓存范围内
        System.out.println(num3 == num4);  // false，因为200超出缓存范围
        
        // 使用equals比较
        System.out.println(num1.equals(num2));  // true
        System.out.println(num3.equals(num4));  // true
        
        // 基本类型与包装类比较
        int primitive = 100;
        System.out.println(num1 == primitive);  // true，包装类会自动拆箱
    }
}

包装类的compareTo方法

所有包装类都实现了Comparable接口，可以使用compareTo方法进行比较：

public class CompareToDemo {
    public static void main(String[] args) {
        Integer num1 = 100;
        Integer num2 = 200;
        
        int result = num1.compareTo(num2);
        if (result < 0) {
            System.out.println("num1 < num2");
        } else if (result > 0) {
            System.out.println("num1 > num2");
        } else {
            System.out.println("num1 == num2");
        }
    }
}

包装类的应用场景

1. 集合类中使用

Java集合不能存储基本类型，只能存储对象，因此需要使用包装类。

import java.util.ArrayList;
import java.util.List;

public class CollectionDemo {
    public static void main(String[] args) {
        // 使用包装类
        List<Integer> numbers = new ArrayList<>();
        numbers.add(10);  // 自动装箱
        numbers.add(20);
        
        int sum = 0;
        for (Integer number : numbers) {
            sum += number;  // 自动拆箱
        }
        System.out.println("Sum: " + sum);
    }
}

2. 泛型中使用

Java泛型不支持基本类型，必须使用包装类。

public class GenericDemo<T> {
    private T value;
    
    public GenericDemo(T value) {
        this.value = value;
    }
    
    public T getValue() {
        return value;
    }
    
    public static void main(String[] args) {
        // 使用Integer，不能使用int
        GenericDemo<Integer> demo = new GenericDemo<>(100);
        Integer value = demo.getValue();
        System.out.println(value);
    }
}

3. 表示null值

基本类型不能为null，而包装类可以。

public class NullDemo {
    public static void main(String[] args) {
        Integer num = null;  // 合法
        // int primitive = null;  // 编译错误
        
        // 使用包装类表示可能为空的值
        Integer result = findValue();
        if (result != null) {
            System.out.println("Found: " + result);
        } else {
            System.out.println("Not found");
        }
    }
    
    public static Integer findValue() {
        // 模拟查找操作，可能返回null
        return Math.random() > 0.5 ? 100 : null;
    }
}

4. 使用包装类的工具方法

包装类提供了许多实用的静态方法。

public class UtilityMethodsDemo {
    public static void main(String[] args) {
        // 字符串转数字
        int num1 = Integer.parseInt("100");
        double num2 = Double.parseDouble("3.14");
        
        // 进制转换
        int decimal = Integer.parseInt("1010", 2);  // 二进制转十进制
        String hex = Integer.toHexString(255);      // 十进制转十六进制
        
        // 数值操作
        int max = Integer.max(10, 20);
        int min = Integer.min(10, 20);
    }
}

包装类的性能考虑

自动装箱/拆箱的性能影响

频繁的自动装箱和拆箱会创建大量临时对象，可能导致性能问题。

public class PerformanceDemo {
    public static void main(String[] args) {
        long start = System.currentTimeMillis();
        
        // 使用包装类
        Integer sum1 = 0;
        for (int i = 0; i < 10000000; i++) {
            sum1 += i;  // 每次循环都会自动装箱和拆箱
        }
        
        long middle = System.currentTimeMillis();
        
        // 使用基本类型
        int sum2 = 0;
        for (int i = 0; i < 10000000; i++) {
            sum2 += i;  // 直接使用基本类型，没有装箱/拆箱开销
        }
        
        long end = System.currentTimeMillis();
        
        System.out.println("使用包装类耗时: " + (middle - start) + "ms");
        System.out.println("使用基本类型耗时: " + (end - middle) + "ms");
    }
}

内存占用

包装类对象比基本类型占用更多内存。

  • int: 4字节
  • Integer: 16字节（对象头8字节 + int值4字节 + 对齐填充4字节）

优化建议

  1. 在性能敏感的场景，优先使用基本类型
  2. 避免在循环中频繁装箱/拆箱
  3. 合理利用缓存机制
  4. 使用基本类型数组而非包装类数组（如int[]而非Integer[]）

常见面试题

1. Integer a = 127, b = 127, c = 128, d = 128，a==b和c==d的结果分别是什么？

Integer a = 127;
Integer b = 127;
Integer c = 128;
Integer d = 128;

System.out.println(a == b);  // true，因为127在缓存范围内
System.out.println(c == d);  // false，因为128超出缓存范围

2. Integer a = new Integer(1)和Integer b = 1有什么区别？

Integer a = new Integer(1);  // 使用构造方法，总是创建新对象
Integer b = 1;               // 自动装箱，使用valueOf，可能使用缓存

System.out.println(a == b);  // false
System.out.println(a.equals(b));  // true

3. 以下代码的输出结果是什么？

Integer a = 1;
Integer b = 2;
Integer c = 3;
Integer d = 3;
Integer e = 321;
Integer f = 321;

System.out.println(c == d);  // true
System.out.println(e == f);  // false
System.out.println(c == (a + b));  // true
System.out.println(c.equals(a + b));  // true

解释：

  • c == d为true，因为3在缓存范围内
  • e == f为false，因为321超出缓存范围
  • c == (a + b)为true，因为a+b会先拆箱计算得到基本类型3，然后与c比较时c也会拆箱，所以是基本类型3==3
  • c.equals(a + b)为true，因为equals比较的是值

4. 下面代码会输出什么？

Double a = 1.0;
Double b = 1.0;
Double c = 2.0 - 1.0;

System.out.println(a == b);  // false，Double没有缓存
System.out.println(a == c);  // false，Double没有缓存
System.out.println(a.equals(b));  // true，值相等
System.out.println(a.equals(c));  // true，值相等

5. 以下代码的输出结果是什么？

public static void main(String[] args) {
    Integer a = 1;
    Integer b = 2;
    swap(a, b);
    System.out.println("a = " + a + ", b = " + b);
}

public static void swap(Integer a, Integer b) {
    Integer temp = a;
    a = b;
    b = temp;
}

输出：a = 1, b = 2

解释：Java是值传递，对于对象是传递引用的副本。在swap方法中，a和b是局部变量，修改它们不会影响main方法中的变量。此外，Integer是不可变类，其值不能被修改。

总结

包装类是Java中基本类型的对象包装，它们提供了许多实用的方法，并使基本类型能够在对象世界中使用。理解包装类的特性、自动装箱/拆箱机制、缓存机制以及性能影响，对于编写高效、正确的Java代码至关重要。

记住以下关键点：

  1. 包装类将基本类型包装成对象
  2. 自动装箱/拆箱是编译器提供的语法糖
  3. 包装类有缓存机制，但范围有限
  4. 使用equals()而非==比较包装类对象的值
  5. 在性能敏感场景优先使用基本类型
  6. 包装类是不可变的，一旦创建值不能修改
