深刻理解Java之Set集合

什么是Set集合

Set集合是Java集合框架中的一个接口，它继承自Collection接口。Set集合就像现实生活中的”集合”概念一样，比如一个班级里的学生集合，每个学生只能出现一次。

想象一下，你有一个装水果的篮子，但这个篮子有个特点：每种水果只能放一个。你不能在篮子里同时放两个苹果或两个橙子。这就是Set集合的核心特性——不允许重复元素。

在Java中，Set接口位于java.util包中，是Collection接口的子接口。它的定义如下：

public interface Set<E> extends Collection<E> {
    // 集合框架中的方法
}

Set接口的特点

  1. 不允许重复元素：Set集合中的元素都是唯一的，不会出现重复的元素。
  2. 无序集合：大多数Set实现类不保证元素的顺序（LinkedHashSet和TreeSet除外）。
  3. 不支持索引访问：不能通过索引来访问Set中的元素。
  4. 最多只允许一个null元素：大多数Set实现允许存储null元素（TreeSet除外）。

Set接口的主要实现类

HashSet

HashSet是Set接口最常用的实现类，它基于HashMap实现。就像一个没有特定顺序的抽奖箱，你放入的元素可能以任意顺序排列。

特点：

  • 不保证元素的顺序
  • 允许使用null元素
  • 非同步（线程不安全）
  • 查找、添加和删除操作的时间复杂度为O(1)

内部实现：
HashSet内部使用HashMap来存储元素。实际上，HashSet是通过将元素作为HashMap的键来实现的，而HashMap的值则是一个固定的Object对象。

// HashSet的部分源码简化版
public class HashSet<E> extends AbstractSet<E> implements Set<E> {
    private transient HashMap<E,Object> map;
    private static final Object PRESENT = new Object();
    
    public HashSet() {
        map = new HashMap<>();
    }
    
    public boolean add(E e) {
        return map.put(e, PRESENT) == null;
    }
}

LinkedHashSet

LinkedHashSet是HashSet的子类，它在HashSet的基础上增加了元素的有序性。想象一个有序的抽奖箱，你按照放入的顺序取出元素。

特点：

  • 维护元素的插入顺序
  • 允许使用null元素
  • 非同步（线程不安全）
  • 性能略低于HashSet，但迭代访问更快

内部实现：
LinkedHashSet内部使用LinkedHashMap来存储元素，这使得它能够记住元素的插入顺序。

TreeSet

TreeSet是Java集合框架中的一个实现类，它是基于TreeMap实现的有序集合。想象一下图书馆的书架，所有书籍都按照特定的规则（如书名字母顺序）排列整齐，这就像TreeSet中的元素一样。它通过红黑树数据结构提供了高效的排序和查找功能。

特点：

  • 元素按照自然顺序或指定的比较器排序
  • 不允许使用null元素
  • 非同步（线程不安全）
  • 查找、添加和删除操作的时间复杂度为O(log n)

内部实现：
TreeSet内部使用TreeMap来存储元素，这使得它能够保持元素的有序性。

TreeSet的基本用法

import java.util.TreeSet;

public class TreeSetDemo {
    public static void main(String[] args) {
        // 创建TreeSet
        TreeSet<Integer> numbers = new TreeSet<>();
        
        // 添加元素
        numbers.add(10);
        numbers.add(5);
        numbers.add(20);
        numbers.add(15);
        numbers.add(5);  // 重复元素不会被添加
        
        // 遍历TreeSet（已排序）
        System.out.println("TreeSet元素（升序）：");
        for (Integer number : numbers) {
            System.out.println(number);
        }
        
        // 输出：5, 10, 15, 20
    }
}

TreeSet的自然排序

Comparable接口

TreeSet默认使用元素的自然排序规则，这要求元素实现Comparable接口。就像图书馆里的书籍可能按照ISBN编号自然排序一样。

import java.util.TreeSet;

// 实现Comparable接口的类
class Book implements Comparable<Book> {
    private String title;
    private double price;
    
    public Book(String title, double price) {
        this.title = title;
        this.price = price;
    }
    
    @Override
    public int compareTo(Book other) {
        // 按书名字母顺序排序
        return this.title.compareTo(other.title);
    }
    
    @Override
    public String toString() {
        return "Book{title='" + title + "', price=" + price + "}";
    }
}

public class ComparableDemo {
    public static void main(String[] args) {
        TreeSet<Book> bookSet = new TreeSet<>();
        
        bookSet.add(new Book("Java编程", 59.9));
        bookSet.add(new Book("Python基础", 45.5));
        bookSet.add(new Book("算法导论", 88.0));
        bookSet.add(new Book("数据结构", 55.0));
        
        // 遍历（按书名字母顺序）
        for (Book book : bookSet) {
            System.out.println(book);
        }
    }
}

compareTo方法的返回值含义

compareTo方法返回一个整数，其含义如下：

  • 返回正数：表示第一个对象（e1）应该排在第二个对象（e2）之后，交互顺序。
  • 返回负数：表示第一个对象（e1）应该排在第二个对象（e2）之前。
  • 返回零：表示两个对象相等，在排序中它们的顺序可以是任意的。

这就像我们比较两本书的ISBN号，如果当前书的ISBN小于另一本，返回负数；相等返回零；大于则返回正数。

TreeSet的定制排序

Comparator接口

如果我们想要使用不同于自然排序的规则，或者元素类没有实现Comparable接口，我们可以使用Comparator接口提供定制排序规则。这就像图书馆可以根据需要改变书籍的排序方式，有时按书名，有时按出版日期。

import java.util.Comparator;
import java.util.TreeSet;

class Student {
    private String name;
    private int age;
    private double score;
    
    public Student(String name, int age, double score) {
        this.name = name;
        this.age = age;
        this.score = score;
    }
    
    public String getName() { return name; }
    public int getAge() { return age; }
    public double getScore() { return score; }
    
    @Override
    public String toString() {
        return "Student{name='" + name + "', age=" + age + ", score=" + score + "}";
    }
}

public class ComparatorDemo {
    public static void main(String[] args) {
        // 子类（这里是匿名内部类）实现Comparator接口，重写compare方法，作为参数传给TreeSet
        // 按年龄排序
        TreeSet<Student> studentsByAge = new TreeSet<>(new Comparator<Student>() {
            @Override
            public int compare(Student s1, Student s2) {
                return s1.getAge() - s2.getAge();
            }
        });
        
        // 使用Lambda表达式按分数排序（降序）
        TreeSet<Student> studentsByScore = new TreeSet<>((s1, s2) -> 
            Double.compare(s2.getScore(), s1.getScore()));
        
        // 添加学生
        Student s1 = new Student("张三", 20, 85.5);
        Student s2 = new Student("李四", 18, 92.0);
        Student s3 = new Student("王五", 22, 78.5);
        Student s4 = new Student("赵六", 19, 88.0);
        
        studentsByAge.add(s1);
        studentsByAge.add(s2);
        studentsByAge.add(s3);
        studentsByAge.add(s4);
        
        studentsByScore.add(s1);
        studentsByScore.add(s2);
        studentsByScore.add(s3);
        studentsByScore.add(s4);
        
        System.out.println("按年龄排序（升序）：");
        for (Student s : studentsByAge) {
            System.out.println(s);
        }
        
        System.out.println("\n按分数排序（降序）：");
        for (Student s : studentsByScore) {
            System.out.println(s);
        }
    }
}

TreeSet排序的内部原理

红黑树实现

TreeSet内部使用红黑树（Red-Black Tree）数据结构来存储元素。红黑树是一种自平衡的二叉搜索树，它通过一系列的规则保证树的高度平衡，从而保证基本操作的时间复杂度为O(log n)。

想象一下一棵树，每个节点都有两个子节点（左子节点和右子节点）。对于树中的任意节点：

  • 左子树上的所有节点都小于当前节点
  • 右子树上的所有节点都大于当前节点

这种结构使得查找、插入和删除操作非常高效。

元素添加过程

当向TreeSet添加元素时，会发生以下步骤：

  1. 使用比较器（或元素的自然顺序）确定新元素的位置
  2. 如果集合为空，新元素成为根节点
  3. 否则，从根节点开始，与树中的节点进行比较
  4. 如果新元素小于当前节点，向左子树移动
  5. 如果新元素大于当前节点，向右子树移动
  6. 如果新元素等于当前节点，根据TreeSet的实现，可能替换现有元素或忽略新元素
  7. 找到合适的位置后，插入新元素
  8. 插入后，可能需要进行树的再平衡操作
// 简化的添加过程伪代码
void add(E element) {
    if (root == null) {
        root = new Node(element);
        return;
    }
    
    Node current = root;
    while (true) {
        int cmp = compare(element, current.element);
        
        if (cmp < 0) {
            if (current.left == null) {
                current.left = new Node(element);
                rebalance();
                return;
            }
            current = current.left;
        } else if (cmp > 0) {
            if (current.right == null) {
                current.right = new Node(element);
                rebalance();
                return;
            }
            current = current.right;
        } else {
            // 元素已存在，根据实现可能替换或忽略
            return;
        }
    }
}

TreeSet排序的常见问题

1. 元素未实现Comparable接口

如果元素没有实现Comparable接口，并且没有提供Comparator，将抛出ClassCastException。

class Person {
    private String name;
    
    public Person(String name) {
        this.name = name;
    }
    
    @Override
    public String toString() {
        return "Person{name='" + name + "'}";
    }
}

public class NoComparableDemo {
    public static void main(String[] args) {
        TreeSet<Person> personSet = new TreeSet<>();
        
        // 将抛出ClassCastException
        personSet.add(new Person("张三"));
    }
}

2. 不一致的比较逻辑

如果compareTo方法或compare方法的实现与equals方法不一致，可能导致TreeSet行为异常。

class InconsistentPerson implements Comparable<InconsistentPerson> {
    private String name;
    private int id;
    
    public InconsistentPerson(String name, int id) {
        this.name = name;
        this.id = id;
    }
    
    @Override
    public int compareTo(InconsistentPerson other) {
        // 只按id比较
        return this.id - other.id;
    }
    
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (obj == null || getClass() != obj.getClass()) return false;
        
        InconsistentPerson that = (InconsistentPerson) obj;
        
        // 按id和name比较
        return id == that.id && name.equals(that.name);
    }
    
    @Override
    public int hashCode() {
        return 31 * name.hashCode() + id;
    }
    
    @Override
    public String toString() {
        return "Person{name='" + name + "', id=" + id + "}";
    }
}

3. 修改已添加元素的关键属性

如果在将元素添加到TreeSet后修改了影响排序的属性，TreeSet的顺序将不再正确。

class MutablePerson implements Comparable<MutablePerson> {
    private String name;
    private int age;
    
    public MutablePerson(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    public void setAge(int age) {
        this.age = age;
    }
    
    @Override
    public int compareTo(MutablePerson other) {
        return this.age - other.age;
    }
    
    @Override
    public String toString() {
        return "MutablePerson{name='" + name + "', age=" + age + "}";
    }
}

public class MutableElementDemo {
    public static void main(String[] args) {
        TreeSet<MutablePerson> personSet = new TreeSet<>();
        
        MutablePerson p1 = new MutablePerson("张三", 20);
        MutablePerson p2 = new MutablePerson("李四", 25);
        MutablePerson p3 = new MutablePerson("王五", 30);
        
        personSet.add(p1);
        personSet.add(p2);
        personSet.add(p3);
        
        System.out.println("修改前：" + personSet);
        
        // 修改p1的年龄，使其大于p3
        p1.setAge(35);
        
        System.out.println("修改后：" + personSet);
        // 顺序不会自动调整！
        
        // 验证顺序错误
        System.out.println("p1是否大于p3: " + (p1.compareTo(p3) > 0));
        System.out.println("但在TreeSet中p1仍在p3之前！");
    }
}

TreeSet与其他集合的对比

TreeSet vs HashSet

特性 TreeSet HashSet
排序 有序 无序
性能(添加/删除/包含) O(log n) O(1)
内部实现 红黑树 哈希表
null元素 不允许 允许一个
使用场景 需要排序的场景 不关心顺序，追求性能

就像图书馆（TreeSet）和仓库（HashSet）的区别：图书馆中的书籍按特定顺序排列，方便查找特定范围的书籍；而仓库中的物品可能没有特定顺序，但可以通过编号快速找到特定物品。

TreeSet vs LinkedHashSet

特性 TreeSet LinkedHashSet
排序 按比较规则排序 按插入顺序排序
性能(添加/删除/包含) O(log n) O(1)
内部实现 红黑树 哈希表+双向链表
使用场景 需要按特定规则排序 需要保持插入顺序

这就像图书馆（TreeSet）和博物馆的展览（LinkedHashSet）：图书馆的书籍按分类和编号排序，而博物馆的展品按参观路线（插入顺序）排列。

TreeSet的性能分析

时间复杂度

操作 时间复杂度
add() O(log n)
remove() O(log n)
contains() O(log n)
size() O(1)
iterator() O(log n)

空间复杂度

TreeSet的空间复杂度为O(n)，其中n是元素数量。

TreeSet的最佳实践

1. 选择合适的排序方式

根据业务需求选择自然排序（Comparable）或定制排序（Comparator）：

  • 如果类有明确的自然顺序，实现Comparable接口
  • 如果需要多种排序方式，或者无法修改类的源代码，使用Comparator

2. 确保比较逻辑与equals一致

为避免意外行为，确保compareTo或compare方法的逻辑与equals方法一致。如果两个对象通过equals方法比较相等，它们的compareTo方法应该返回0。

3. 避免修改影响排序的属性

将TreeSet中的元素视为不可变对象，不要修改影响排序的属性。如果必须修改，考虑：

  • 先从TreeSet中移除元素
  • 修改属性
  • 重新添加到TreeSet中
TreeSet<MutablePerson> personSet = new TreeSet<>();
MutablePerson person = new MutablePerson("张三", 20);
personSet.add(person);

// 需要修改年龄时
personSet.remove(person);
person.setAge(25);
personSet.add(person);

4. 利用TreeSet的导航方法

TreeSet提供了丰富的导航方法，可以高效地查找特定范围的元素：

TreeSet<Integer> numbers = new TreeSet<>();
for (int i = 1; i <= 100; i++) {
    numbers.add(i);
}

// 获取小于或等于指定元素的最大元素
System.out.println(numbers.floor(50));  // 50

// 获取大于或等于指定元素的最小元素
System.out.println(numbers.ceiling(50.5));  // 51

// 获取严格小于指定元素的最大元素
System.out.println(numbers.lower(50));  // 49

// 获取严格大于指定元素的最小元素
System.out.println(numbers.higher(50));  // 51

// 获取指定范围的子集
SortedSet<Integer> subset = numbers.subSet(20, 30);
System.out.println(subset);  // [20, 21, ..., 29]

5. 考虑线程安全

TreeSet不是线程安全的。在多线程环境中，可以使用以下方式确保线程安全：

// 使用Collections.synchronizedSortedSet包装
SortedSet<String> sortedSet = Collections.synchronizedSortedSet(new TreeSet<>());

// 或使用并发集合类
import java.util.concurrent.ConcurrentSkipListSet;
ConcurrentSkipListSet<String> concurrentSet = new ConcurrentSkipListSet<>();

6. 性能考虑

  • 对于频繁查找操作但不关心顺序的场景，考虑使用HashSet
  • 对于需要按插入顺序访问的场景，考虑使用LinkedHashSet
  • 只有在需要元素有序或需要范围查询时，才使用TreeSet

EnumSet

EnumSet是专门为枚举类型设计的Set实现。如果你的集合只包含枚举类型的元素，EnumSet是最佳选择。

特点：

  • 所有元素必须来自同一个枚举类型
  • 内部使用位向量实现，非常高效
  • 不允许使用null元素
  • 元素按照枚举常量的声明顺序排序

使用示例：

enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY }

EnumSet<Day> weekdays = EnumSet.range(Day.MONDAY, Day.FRIDAY);
EnumSet<Day> weekend = EnumSet.complementOf(weekdays); // 包含SATURDAY和SUNDAY

CopyOnWriteArraySet

CopyOnWriteArraySet是线程安全的Set实现，适用于读多写少的并发场景。

特点：

  • 线程安全，无需额外同步
  • 在修改时复制整个底层数组，适合读多写少的场景
  • 迭代器不支持修改操作
  • 允许使用null元素

内部实现：
CopyOnWriteArraySet内部使用CopyOnWriteArrayList来存储元素。

Set集合的应用场景

Set集合在实际开发中有很多应用场景，以下是一些常见的：

  1. 去重：当需要去除重复元素时，可以使用Set集合。

    List<Integer> numbersList = Arrays.asList(1, 2, 3, 2, 1, 4, 5, 4);
Set<Integer> numbersSet = new HashSet<>(numbersList); // 结果为 [1, 2, 3, 4, 5]

  2. 检查元素是否存在：Set的contains方法时间复杂度为O(1)（对于HashSet），比List的contains方法更高效。

    Set<String> dictionary = new HashSet<>();
dictionary.add("apple");
dictionary.add("banana");

boolean exists = dictionary.contains("apple"); // true

  3. 集合运算：可以使用Set的方法实现并集、交集、差集等集合运算。

    Set<Integer> set1 = new HashSet<>(Arrays.asList(1, 2, 3, 4));
Set<Integer> set2 = new HashSet<>(Arrays.asList(3, 4, 5, 6));

// 并集
Set<Integer> union = new HashSet<>(set1);
union.addAll(set2); // [1, 2, 3, 4, 5, 6]

// 交集
Set<Integer> intersection = new HashSet<>(set1);
intersection.retainAll(set2); // [3, 4]

// 差集
Set<Integer> difference = new HashSet<>(set1);
difference.removeAll(set2); // [1, 2]

  4. 有序集合：当需要元素保持特定顺序时，可以使用TreeSet或LinkedHashSet。

    // 按照自然顺序排序
Set<String> sortedSet = new TreeSet<>();
sortedSet.add("banana");
sortedSet.add("apple");
sortedSet.add("orange");
// 结果为 ["apple", "banana", "orange"]

// 按照插入顺序排序
Set<String> orderedSet = new LinkedHashSet<>();
orderedSet.add("banana");
orderedSet.add("apple");
orderedSet.add("orange");
// 结果为 ["banana", "apple", "orange"]

  5. 缓存实现：可以使用LinkedHashSet实现LRU（最近最少使用）缓存。

  6. 线程安全的集合：在多线程环境下，可以使用CopyOnWriteArraySet或Collections.synchronizedSet()方法包装的Set。

常见问题与最佳实践

1. 如何正确地在Set中存储自定义对象？

当在Set中存储自定义对象时，需要正确地重写equals()和hashCode()方法，否则Set无法正确地判断元素是否重复。

public class Person {
    private String name;
    private int age;
    
    // 构造函数、getter和setter省略
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Person person = (Person) o;
        return age == person.age && Objects.equals(name, person.name);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }
}

2. 如何选择合适的Set实现类？

  • 如果不需要有序性，且需要最高效率，选择HashSet
  • 如果需要保持插入顺序，选择LinkedHashSet
  • 如果需要元素自动排序，选择TreeSet
  • 如果只存储枚举类型，选择EnumSet
  • 如果在多线程环境下使用，选择CopyOnWriteArraySet或使用Collections.synchronizedSet()包装其他Set实现

3. 避免在遍历Set时修改集合

在使用for-each循环或迭代器遍历Set时，不应该直接调用Set的remove()方法来删除元素，这会导致ConcurrentModificationException异常。正确的做法是使用迭代器的remove()方法：

Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
set.add("Orange");

// 错误的做法
// for (String fruit : set) {
//     if ("Banana".equals(fruit)) {
//         set.remove(fruit); // 会抛出ConcurrentModificationException异常
//     }
// }

// 正确的做法
Iterator<String> iterator = set.iterator();
while (iterator.hasNext()) {
    String fruit = iterator.next();
    if ("Banana".equals(fruit)) {
        iterator.remove(); // 使用迭代器的remove方法
    }
}

4. 使用Stream API简化Set操作

Java 8引入的Stream API可以大大简化Set的操作：

Set<String> set = new HashSet<>(Arrays.asList("Apple", "Banana", "Orange", "Grape"));

// 过滤元素
Set<String> filteredSet = set.stream()
                             .filter(fruit -> fruit.length() > 5)
                             .collect(Collectors.toSet());
// 结果为 ["Banana", "Orange"]

// 转换元素
Set<String> upperCaseSet = set.stream()
                             .map(String::toUpperCase)
                             .collect(Collectors.toSet());
// 结果为 ["APPLE", "BANANA", "ORANGE", "GRAPE"]

示例3：集合运算

import java.util.*;

public class SetOperations {
    public static void main(String[] args) {
        Set<Integer> set1 = new HashSet<>(Arrays.asList(1, 2, 3, 4, 5));
        Set<Integer> set2 = new HashSet<>(Arrays.asList(4, 5, 6, 7, 8));
        
        // 打印原始集合
        System.out.println("Set1: " + set1);
        System.out.println("Set2: " + set2);
        
        // 并集
        Set<Integer> union = new HashSet<>(set1);
        union.addAll(set2);
        System.out.println("Union: " + union);
        
        // 交集
        Set<Integer> intersection = new HashSet<>(set1);
        intersection.retainAll(set2);
        System.out.println("Intersection: " + intersection);
        
        // 差集 (set1 - set2)
        Set<Integer> difference1 = new HashSet<>(set1);
        difference1.removeAll(set2);
        System.out.println("Difference (set1 - set2): " + difference1);
        
        // 差集 (set2 - set1)
        Set<Integer> difference2 = new HashSet<>(set2);
        difference2.removeAll(set1);
        System.out.println("Difference (set2 - set1): " + difference2);
        
        // 对称差集 (set1 ∆ set2)
        Set<Integer> symmetricDifference = new HashSet<>(set1);
        symmetricDifference.addAll(set2); // 并集
        Set<Integer> temp = new HashSet<>(set1);
        temp.retainAll(set2); // 交集
        symmetricDifference.removeAll(temp); // 并集 - 交集 = 对称差集
        System.out.println("Symmetric Difference: " + symmetricDifference);
    }
}

import java.util.*;
import java.util.stream.Collectors;

public class SetStreamDemo {
    public static void main(String[] args) {
        Set<String> fruits = new HashSet<>(Arrays.asList(
            "Apple", "Banana", "Orange", "Grape", "Watermelon", "Kiwi"
        ));
        
        System.out.println("Original Set: " + fruits);
        
        // 过滤长度大于5的水果
        Set<String> longNameFruits = fruits.stream()
                                          .filter(fruit -> fruit.length() > 5)
                                          .collect(Collectors.toSet());
        System.out.println("Fruits with name length > 5: " + longNameFruits);
        
        // 转换为大写
        Set<String> upperCaseFruits = fruits.stream()
                                           .map(String::toUpperCase)
                                           .collect(Collectors.toSet());
        System.out.println("Uppercase fruits: " + upperCaseFruits);
        
        // 按长度分组
        Map<Integer, List<String>> fruitsByLength = fruits.stream()
                                                         .collect(Collectors.groupingBy(String::length));
        System.out.println("Fruits grouped by length: " + fruitsByLength);
        
        // 检查是否所有水果名称都以'A'开头
        boolean allStartWithA = fruits.stream().allMatch(fruit -> fruit.startsWith("A"));
        System.out.println("All fruits start with 'A': " + allStartWithA);
        
        // 检查是否有水果名称以'A'开头
        boolean anyStartWithA = fruits.stream().anyMatch(fruit -> fruit.startsWith("A"));
        System.out.println("Any fruit starts with 'A': " + anyStartWithA);
        
        // 获取名称最长的水果
        Optional<String> longestFruit = fruits.stream()
                                             .max(Comparator.comparingInt(String::length));
        longestFruit.ifPresent(fruit -> System.out.println("Longest fruit name: " + fruit));
    }
}
