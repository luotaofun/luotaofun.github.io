详解Java异常处理机制

发表于 2025-03-13

详解Java异常处理机制

Java提供了一种结构化的方式来处理程序运行时可能出现的错误情况。

异常的基本概念

什么是异常

异常是程序运行过程中出现的非正常情况，如：

  • 程序逻辑错误
  • 系统资源不足
  • 用户操作错误
  • 设备故障等

Java使用异常对象（Exception Object）来表示这些异常情况，并提供了处理这些异常的机制。

异常的分类

Java中的异常主要分为三大类：

  1. 检查型异常（Checked Exception）

    • 编译时异常，必须显式处理
    • 继承自Exception类但不是RuntimeException的子类
    • 例如：IOException, SQLException

  2. 非检查型异常（Unchecked Exception）

    • 运行时异常，不强制处理
    • 继承自RuntimeException
    • 例如：NullPointerException, ArrayIndexOutOfBoundsException

  3. 错误（Error）

    • 表示严重问题，通常无法恢复
    • 继承自Error类
    • 例如：OutOfMemoryError, StackOverflowError

Java异常类层次结构

Throwable (所有异常和错误的父类)
├── Error (严重错误，通常无法恢复)
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── ...
└── Exception (异常情况，可以处理)
    ├── IOException (检查型异常)
    ├── SQLException (检查型异常)
    ├── ClassNotFoundException (检查型异常)
    ├── RuntimeException (非检查型异常)
    │   ├── NullPointerException
    │   ├── ArrayIndexOutOfBoundsException
    │   ├── ArithmeticException
    │   ├── IllegalArgumentException
    │   └── ...
    └── ...

异常处理语法

try-catch-finally

基本语法：

try {
    // 可能抛出异常的代码
} catch (ExceptionType1 e1) {
    // 处理ExceptionType1类型的异常
} catch (ExceptionType2 e2) {
    // 处理ExceptionType2类型的异常
} finally {
    // 无论是否发生异常，都会执行的代码
    // 通常用于释放资源
}

示例：

public void readFile(String filename) {
    FileInputStream fis = null;
    try {
        fis = new FileInputStream(filename);
        int data;
        while ((data = fis.read()) != -1) {
            System.out.print((char) data);
        }
    } catch (FileNotFoundException e) {
        System.err.println("文件不存在: " + e.getMessage());
    } catch (IOException e) {
        System.err.println("读取文件错误: " + e.getMessage());
    } finally {
        // 关闭资源
        if (fis != null) {
            try {
                fis.close();
            } catch (IOException e) {
                System.err.println("关闭文件错误: " + e.getMessage());
            }
        }
    }
}
try-with-resources (Java 7+)

自动资源管理，简化了资源的关闭操作：

public void readFile(String filename) {
    // 实现了AutoCloseable接口的资源会在try块结束后自动关闭
    try (FileInputStream fis = new FileInputStream(filename)) {
        int data;
        while ((data = fis.read()) != -1) {
            System.out.print((char) data);
        }
    } catch (FileNotFoundException e) {
        System.err.println("文件不存在: " + e.getMessage());
    } catch (IOException e) {
        System.err.println("读取文件错误: " + e.getMessage());
    }
    // 不需要finally块来关闭资源
}

多个资源的情况：

try (FileInputStream fis = new FileInputStream("input.txt");
     FileOutputStream fos = new FileOutputStream("output.txt")) {
    // 使用fis和fos
} catch (IOException e) {
    // 处理异常
}
多重捕获 (Java 7+)

一个catch块可以处理多种类型的异常：

try {
    // 可能抛出异常的代码
} catch (FileNotFoundException | SQLException e) {
    // 处理FileNotFoundException或SQLException
    System.err.println("发生异常: " + e.getMessage());
}
throws关键字

用于声明方法可能抛出的检查型异常：

public void readFile(String filename) throws IOException {
    FileInputStream fis = new FileInputStream(filename); // 可能抛出FileNotFoundException
    int data;
    while ((data = fis.read()) != -1) { // 可能抛出IOException
        System.out.print((char) data);
    }
    fis.close(); // 可能抛出IOException
}

调用带throws声明的方法：

public void processFile(String filename) {
    try {
        readFile(filename);
    } catch (IOException e) {
        System.err.println("处理文件错误: " + e.getMessage());
    }
}
throw关键字

用于手动抛出异常：

public void validateAge(int age) {
    if (age < 0) {
        throw new IllegalArgumentException("年龄不能为负数");
    }
    if (age > 150) {
        throw new IllegalArgumentException("年龄不合理");
    }
    System.out.println("年龄验证通过: " + age);
}

自定义异常

当标准异常不能满足需求时，可以创建自定义异常：

// 自定义检查型异常
public class InsufficientFundsException extends Exception {
    private double amount;
    
    public InsufficientFundsException(double amount) {
        super("余额不足，还差: " + amount);
        this.amount = amount;
    }
    
    public double getAmount() {
        return amount;
    }
}

// 自定义运行时异常
public class InvalidUserException extends RuntimeException {
    public InvalidUserException(String message) {
        super(message);
    }
}

使用自定义异常：

public class BankAccount {
    private double balance;
    
    public void withdraw(double amount) throws InsufficientFundsException {
        if (amount > balance) {
            throw new InsufficientFundsException(amount - balance);
        }
        balance -= amount;
    }
    
    public void validateUser(String username) {
        if (username == null || username.trim().isEmpty()) {
            throw new InvalidUserException("用户名不能为空");
        }
    }
}

异常链

异常链允许将一个异常包装在另一个异常中，保留原始异常信息：

try {
    // 数据库操作
} catch (SQLException e) {
    // 包装原始异常，转换为应用层异常
    throw new ServiceException("数据库操作失败", e);
}

常见异常及其处理方法

NullPointerException
// 避免NullPointerException
public void processString(String str) {
    // 不好的方式
    int length = str.length(); // 如果str为null，抛出NullPointerException
    
    // 好的方式
    if (str != null) {
        int length = str.length();
    }
    
    // 更好的方式 (Java 8+)
    Optional<String> optStr = Optional.ofNullable(str);
    optStr.ifPresent(s -> {
        int length = s.length();
    });
}
ArrayIndexOutOfBoundsException
// 避免ArrayIndexOutOfBoundsException
public void accessArray(int[] array, int index) {
    // 不好的方式
    int value = array[index]; // 如果index超出范围，抛出异常
    
    // 好的方式
    if (array != null && index >= 0 && index < array.length) {
        int value = array[index];
    }
}
NumberFormatException
// 避免NumberFormatException
public int parseInteger(String str) {
    // 不好的方式
    return Integer.parseInt(str); // 如果str不是有效的数字，抛出异常
    
    // 好的方式
    try {
        return Integer.parseInt(str);
    } catch (NumberFormatException e) {
        System.err.println("无效的数字格式: " + str);
        return 0; // 或其他默认值
    }
}

异常处理的最佳实践

  1. 只捕获能够处理的异常

    // 不好的方式
try {
    // 代码
} catch (Exception e) {
    e.printStackTrace(); // 捕获所有异常但没有适当处理
}

// 好的方式
try {
    // 代码
} catch (IOException e) {
    // 针对IO异常的具体处理
} catch (SQLException e) {
    // 针对SQL异常的具体处理
}

  2. 不要忽略异常

    // 不好的方式
try {
    // 代码
} catch (Exception e) {
    // 空的catch块，完全忽略异常
}

// 好的方式
try {
    // 代码
} catch (Exception e) {
    logger.error("发生错误", e);
    // 适当的错误处理
}

  3. 释放资源的正确方式

    // Java 7之前
FileInputStream fis = null;
try {
    fis = new FileInputStream("file.txt");
    // 使用fis
} catch (IOException e) {
    // 处理异常
} finally {
    if (fis != null) {
        try {
            fis.close();
        } catch (IOException e) {
            // 处理关闭异常
        }
    }
}

// Java 7及以后
try (FileInputStream fis = new FileInputStream("file.txt")) {
    // 使用fis
} catch (IOException e) {
    // 处理异常
}

  4. 异常的粒度

    // 粒度太粗
try {
    // 很多操作
} catch (Exception e) {
    // 处理异常
}

// 适当的粒度
try {
    // 操作1
} catch (SpecificException e) {
    // 处理特定异常
}

try {
    // 操作2
} catch (AnotherException e) {
    // 处理另一种异常
}

  5. 使用标准异常

    // 不好的方式
throw new Exception("参数无效");

// 好的方式
throw new IllegalArgumentException("参数无效");

  6. 记录异常信息

    try {
    // 代码
} catch (Exception e) {
    logger.error("操作失败: " + e.getMessage(), e);
    // 包含异常堆栈信息
}

  7. 异常转换

    try {
    // 底层操作
} catch (LowLevelException e) {
    // 转换为更有意义的高级异常
    throw new BusinessException("业务操作失败", e);
}
