Java多线程与IO学习笔记

一、多线程

1. 什么是多线程

想象一下，你就像一个超级厨师，在厨房里同时做几道菜。单线程就好比你一次只能专注做一道菜，做完一道再做下一道；而多线程呢，就像是你可以同时处理好几道菜，这边炒着菜，那边煮着汤，还能同时切着水果。这样效率是不是就大大提高啦？在计算机里，多线程就是让程序可以同时执行多个任务，充分利用CPU的资源。

2. 多线程的使用方法

在Java里，要创建多线程有两种常见的方法。一种是继承Thread类，就好像你开了一家新的餐厅分店，这个分店有自己独特的经营方式（也就是线程的执行逻辑）。另一种是实现Runnable接口，这就好比你有一个通用的经营方案，很多餐厅都可以按照这个方案来经营（多个线程可以共享同一个任务逻辑）。

继承Thread类示例代码

class MyThread extends Thread {
    public void run() {
        System.out.println("这是一个继承Thread类的线程在执行任务");
    }
}

public class Main {
    public static void main(String[] args) {
        MyThread thread = new MyThread();
        thread.start();
    }
}

实现Runnable接口示例代码

class MyRunnable implements Runnable {
    public void run() {
        System.out.println("这是一个实现Runnable接口的线程在执行任务");
    }
}

public class Main {
    public static void main(String[] args) {
        MyRunnable myRunnable = new MyRunnable();
        Thread thread = new Thread(myRunnable);
        thread.start();
    }
}

3. 多线程的注意事项

多线程虽然能提高效率，但也可能会带来一些问题，就像餐厅同时接待很多客人，可能会出现服务混乱的情况。在多线程里，最常见的问题就是线程安全问题，比如多个线程同时访问和修改同一个数据，就可能会导致数据不一致。所以，我们需要使用一些同步机制来保证线程安全，就像餐厅要制定一些规则来保证服务的有序进行。

二、IO（输入输出）

1. 什么是IO

IO就像是你和外界交流的通道。想象一下，你要从外面的世界获取信息，就像你要从超市里买东西回家，这就是输入；你要把自己的想法告诉别人，就像你要把家里的东西卖给别人，这就是输出。在计算机里，IO就是程序和外部设备（如文件、网络等）之间进行数据交换的过程。

2. IO的分类

Java里的IO可以分为字节流和字符流。字节流就像是一个个小的水滴，它可以处理任何类型的数据，包括图片、音频等；字符流就像是一个个文字，它主要用于处理文本数据。

3. IO的使用方法

文件读取示例（字节流）

import java.io.FileInputStream;
import java.io.IOException;

public class FileReadExample {
    public static void main(String[] args) {
        try (FileInputStream fis = new FileInputStream("test.txt")) {

                System.out.print((char) data);
            }
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

            fos.write(bytes);
        } catch (IOException e) {

深刻理解Java之Map集合

什么是Map集合

Map集合是Java集合框架中一个重要的接口，它用于存储键值对（key-value pairs）。想象一个图书馆的书架系统：每本书都有一个唯一的编号（key），通过这个编号可以找到对应的书（value）。这就是Map的基本工作原理。

生活中的例子：

  • 学生证号 -> 学生信息
  • 身份证号 -> 居民信息
  • 手机号码 -> 联系人
  • 银行卡号 -> 账户信息

在Java中，Map接口位于java.util包中，它的基本定义如下：

public interface Map<K,V> {
    // K 表示键（Key）的类型
    // V 表示值（Value）的类型
}

Map接口的特点

  1. 键值对映射：每个元素都是键值对，包含一个键和一个值
  2. 键的唯一性：键不能重复，但值可以重复
  3. 键值对的存储：键和值都可以是任何引用类型的数据，但一般不建议使用可变对象作为键
  4. 键和null：大多数Map实现允许使用null键（TreeMap除外），但只能有一个null键
  5. 值和null：所有Map实现都允许使用null值，且可以有多个null值

Map接口的主要实现类

HashMap

HashMap是最常用的Map实现类，就像一个超市的储物柜系统：每个柜子都有一个唯一的号码（key），用来存储顾客的物品（value）。

特点：

  • 无序存储
  • 允许null键和null值
  • 非同步（线程不安全）
  • 查找和插入的时间复杂度为O(1)

内部实现：

public class HashMap<K,V> extends AbstractMap<K,V> implements Map<K,V> {
    // 默认初始容量 - 必须是2的幂
    static final int DEFAULT_INITIAL_CAPACITY = 16;
    
    // 默认加载因子
    static final float DEFAULT_LOAD_FACTOR = 0.75f;
    
    // 当链表长度大于此值时，转换为红黑树
    static final int TREEIFY_THRESHOLD = 8;
    
    // Node数组，用于存储键值对
    transient Node<K,V>[] table;
}

LinkedHashMap

LinkedHashMap是HashMap的子类，它在HashMap的基础上维护了一个双向链表，用于记录元素的插入顺序或访问顺序。想象一个有序的超市储物柜，不仅能通过柜号找到物品，还能按存入顺序或使用频率排序。

特点：

  • 维护元素的插入顺序或访问顺序（可选）
  • 适合实现LRU缓存
  • 性能略低于HashMap
  • 允许null键和null值

示例：

// 创建一个记录访问顺序的LinkedHashMap（LRU缓存实现）
LinkedHashMap<String, String> lruCache = new LinkedHashMap<>(16, 0.75f, true) {
    @Override
    protected boolean removeEldestEntry(Map.Entry<String, String> eldest) {
        return size() > 3; // 限制缓存大小为3
    }
};

TreeMap

TreeMap基于红黑树实现，它的键值对总是按照键的自然顺序或指定的比较器排序。想象一个按字母顺序排列的字典，你可以快速找到特定字母开头的词条。

特点：

  • 键值对按键排序（自然顺序或自定义顺序）
  • 不允许null键，但允许null值
  • 查找和插入的时间复杂度为O(log n)
  • 适合需要排序的场景

示例：

// 创建一个按键的长度排序的TreeMap
TreeMap<String, Integer> lengthSortedMap = new TreeMap<>((s1, s2) -> {
    int lenCompare = Integer.compare(s1.length(), s2.length());
    return lenCompare != 0 ? lenCompare : s1.compareTo(s2);
});

Hashtable

Hashtable是一个历史悠久的Map实现，现在主要用于遗留代码。它的所有方法都是同步的，类似于一个带锁的储物柜系统。

特点：

  • 线程安全（所有方法都是同步的）
  • 不允许null键和null值
  • 性能较差（因为同步）
  • 已被ConcurrentHashMap取代

ConcurrentHashMap

ConcurrentHashMap是专门为并发设计的Map实现，它像一个分区的储物柜系统，不同的区域可以同时被不同的人使用。

特点：

  • 线程安全
  • 支持高并发访问
  • 不允许null键和null值
  • 性能优于Hashtable

示例：

ConcurrentHashMap<String, Integer> concurrentMap = new ConcurrentHashMap<>();
// 线程安全的操作
concurrentMap.computeIfAbsent("count", k -> 0);
concurrentMap.computeIfPresent("count", (k, v) -> v + 1);

WeakHashMap

WeakHashMap的特殊之处在于它的键是弱引用，当键对象不再被其他对象引用时，相应的键值对会被自动移除。想象一个会自动清理的储物柜系统。

特点：

  • 键是弱引用
  • 适合实现缓存
  • 自动移除无用的键值对
  • 需要注意垃圾回收的影响

EnumMap

EnumMap是专门为枚举类型设计的Map实现，非常高效。

特点：

  • 键必须是同一个枚举类型
  • 不允许null键
  • 内部使用数组实现，性能极高
  • 保持枚举定义的顺序

常见问题与最佳实践

1. 如何选择合适的Map实现？

  • 一般场景：使用HashMap
  • 需要保持插入顺序：使用LinkedHashMap
  • 需要键排序：使用TreeMap
  • 需要线程安全：使用ConcurrentHashMap
  • 键是枚举类型：使用EnumMap
  • 需要缓存功能：使用WeakHashMap或自定义的LinkedHashMap

2. 自定义对象作为键的注意事项

public class CustomKey {
    private String id;
    private String name;
    
    // 必须正确实现equals和hashCode方法
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        CustomKey customKey = (CustomKey) o;
        return Objects.equals(id, customKey.id) &&
               Objects.equals(name, customKey.name);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(id, name);
    }
}

3. 避免并发修改问题

Map<String, Integer> map = new HashMap<>();
// 错误的方式
for (String key : map.keySet()) {
    if (someCondition) {
        map.remove(key); // 可能抛出ConcurrentModificationException

// 正确的方式
Iterator<Map.Entry<String, Integer>> iterator = map.entrySet().iterator();
while (iterator.hasNext()) {
    Map.Entry<String, Integer> entry = iterator.next();
    if (someCondition) {
        iterator.remove(); // 安全的删除方式
    }
}

import java.util.*;

public class MapDemo {

        // 遍历Map
        for (Map.Entry<String, Integer> entry : scores.entrySet()) {
            System.out.printf("%s scored %d%n", entry.getKey(), entry.getValue());
        }
        
        // 清空Map
        scores.clear();
        System.out.println("Is empty? " + scores.isEmpty());
    }

import java.util.*;

public class AdvancedMapOperations {
    public static void main(String[] args) {
        Map<String, Integer> map = new HashMap<>();
        
        // 使用compute方法
        map.compute("A", (k, v) -> (v == null) ? 1 : v + 1);
        map.compute("A", (k, v) -> v + 1);
        System.out.println("A count: " + map.get("A")); // 输出2
        
        // 使用computeIfAbsent
        map.computeIfAbsent("B", k -> 1);
        map.computeIfAbsent("B", k -> 2); // 不会改变B的值
        System.out.println("B count: " + map.get("B")); // 输出1
        
        // 使用computeIfPresent
        map.computeIfPresent("B", (k, v) -> v + 1);
        System.out.println("B count: " + map.get("B")); // 输出2
        
        // 使用merge方法
        map.merge("C", 1, Integer::sum);
        map.merge("C", 1, Integer::sum);
        System.out.println("C count: " + map.get("C")); // 输出2
        
        // 批量操作
        map.replaceAll((k, v) -> v * 2);
        System.out.println("After doubling all values: " + map);
        
        // 使用Stream API
        int sum = map.values().stream()
                    .mapToInt(Integer::intValue)
                    .sum();
        System.out.println("Sum of all values: " + sum);
        
        // 按条件过滤

什么是Set集合

Set集合是Java集合框架中的一个接口，它继承自Collection接口。Set集合就像现实生活中的”集合”概念一样，比如一个班级里的学生集合，每个学生只能出现一次。

想象一下，你有一个装水果的篮子，但这个篮子有个特点：每种水果只能放一个。你不能在篮子里同时放两个苹果或两个橙子。这就是Set集合的核心特性——不允许重复元素。

在Java中，Set接口位于java.util包中，是Collection接口的子接口。它的定义如下：

public interface Set<E> extends Collection<E> {
    // 集合框架中的方法
}

Set接口的特点

  1. 不允许重复元素：Set集合中的元素都是唯一的，不会出现重复的元素。
  2. 无序集合：大多数Set实现类不保证元素的顺序（LinkedHashSet和TreeSet除外）。
  3. 不支持索引访问：不能通过索引来访问Set中的元素。
  4. 最多只允许一个null元素：大多数Set实现允许存储null元素（TreeSet除外）。

Set接口的主要实现类

HashSet

HashSet是Set接口最常用的实现类，它基于HashMap实现。就像一个没有特定顺序的抽奖箱，你放入的元素可能以任意顺序排列。

特点：

  • 不保证元素的顺序
  • 允许使用null元素
  • 非同步（线程不安全）
  • 查找、添加和删除操作的时间复杂度为O(1)

内部实现：
HashSet内部使用HashMap来存储元素。实际上，HashSet是通过将元素作为HashMap的键来实现的，而HashMap的值则是一个固定的Object对象。

// HashSet的部分源码简化版
public class HashSet<E> extends AbstractSet<E> implements Set<E> {
    private transient HashMap<E,Object> map;
    private static final Object PRESENT = new Object();
    
    public HashSet() {
        map = new HashMap<>();
    }
    
    public boolean add(E e) {
        return map.put(e, PRESENT) == null;
    }
}

LinkedHashSet

LinkedHashSet是HashSet的子类，它在HashSet的基础上增加了元素的有序性。想象一个有序的抽奖箱，你按照放入的顺序取出元素。

特点：

  • 维护元素的插入顺序
  • 允许使用null元素
  • 非同步（线程不安全）
  • 性能略低于HashSet，但迭代访问更快

内部实现：
LinkedHashSet内部使用LinkedHashMap来存储元素，这使得它能够记住元素的插入顺序。

TreeSet

TreeSet基于TreeMap实现，是一个有序的Set集合。想象一个自动排序的抽奖箱，无论你以什么顺序放入元素，取出时总是按照特定顺序（默认是自然顺序）。

特点：

  • 元素按照自然顺序或指定的比较器排序
  • 不允许使用null元素
  • 非同步（线程不安全）
  • 查找、添加和删除操作的时间复杂度为O(log n)

内部实现：
TreeSet内部使用TreeMap来存储元素，这使得它能够保持元素的有序性。

EnumSet

EnumSet是专门为枚举类型设计的Set实现。如果你的集合只包含枚举类型的元素，EnumSet是最佳选择。

特点：

  • 所有元素必须来自同一个枚举类型
  • 内部使用位向量实现，非常高效
  • 不允许使用null元素
  • 元素按照枚举常量的声明顺序排序

使用示例：

enum Day { MONDAY, TUESDAY, WEDNESDAY, THURSDAY, FRIDAY, SATURDAY, SUNDAY }

EnumSet<Day> weekdays = EnumSet.range(Day.MONDAY, Day.FRIDAY);
EnumSet<Day> weekend = EnumSet.complementOf(weekdays); // 包含SATURDAY和SUNDAY

CopyOnWriteArraySet

CopyOnWriteArraySet是线程安全的Set实现，适用于读多写少的并发场景。

特点：

  • 线程安全，无需额外同步
  • 在修改时复制整个底层数组，适合读多写少的场景
  • 迭代器不支持修改操作
  • 允许使用null元素

内部实现：
CopyOnWriteArraySet内部使用CopyOnWriteArrayList来存储元素。

1. 如何正确地在Set中存储自定义对象？

当在Set中存储自定义对象时，需要正确地重写equals()和hashCode()方法，否则Set无法正确地判断元素是否重复。

public class Person {
    private String name;
    private int age;
    
    // 构造函数、getter和setter省略
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Person person = (Person) o;
        return age == person.age && Objects.equals(name, person.name);
    }
    
    @Override
    public int hashCode() {
        return Objects.hash(name, age);
    }
}

2. 如何选择合适的Set实现类？

  • 如果不需要有序性，且需要最高效率，选择HashSet
  • 如果需要保持插入顺序，选择LinkedHashSet
  • 如果需要元素自动排序，选择TreeSet
  • 如果只存储枚举类型，选择EnumSet
  • 如果在多线程环境下使用，选择CopyOnWriteArraySet或使用Collections.synchronizedSet()包装其他Set实现

3. 避免在遍历Set时修改集合

在使用for-each循环或迭代器遍历Set时，不应该直接调用Set的remove()方法来删除元素，这会导致ConcurrentModificationException异常。正确的做法是使用迭代器的remove()方法：

Set<String> set = new HashSet<>();
set.add("Apple");
set.add("Banana");
set.add("Orange");

// 错误的做法
// for (String fruit : set) {
//     if ("Banana".equals(fruit)) {
//         set.remove(fruit); // 会抛出ConcurrentModificationException异常
//     }
// }

// 正确的做法
Iterator<String> iterator = set.iterator();
while (iterator.hasNext()) {
    String fruit = iterator.next();
    if ("Banana".equals(fruit)) {
        iterator.remove(); // 使用迭代器的remove方法
    }
}

2.1 ArrayList - “整齐排列的火车车厢”

ArrayList是List接口最常用的实现类，基于动态数组实现。

内部结构：

// ArrayList的核心是一个Object数组
transient Object[] elementData;

// 实际元素数量
private int size;

工作原理：

  • 初始容量为10（默认构造函数）
  • 当容量不足时，自动扩容为原容量的1.5倍
  • 扩容涉及数组复制，是一个相对昂贵的操作

性能特点：

  • 随机访问性能极佳：O(1)
  • 尾部添加元素通常为O(1)，但可能触发扩容
  • 中间插入/删除元素性能较差：O(n)，因为需要移动元素
  • 非线程安全

生活比喻：
ArrayList就像一列火车车厢，整齐地排列在铁轨上：

  • 想找到第N号车厢非常容易，直接数过去就行（随机访问快）
  • 在中间加入新车厢很麻烦，需要把后面所有车厢都移动（中间插入慢）
  • 在尾部加车厢相对简单，但偶尔需要换一条更长的铁轨（扩容操作）
  • 每个车厢紧挨着，占用空间紧凑（内存效率高）

代码示例：

// 创建ArrayList
List<String> fruits = new ArrayList<>();

// 添加元素
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Orange");

// 指定位置添加
fruits.add(1, "Mango");  // [Apple, Mango, Banana, Orange]

// 访问元素
String fruit = fruits.get(0);  // Apple

// 修改元素
fruits.set(0, "Pineapple");  // [Pineapple, Mango, Banana, Orange]

// 删除元素
fruits.remove(2);  // [Pineapple, Mango, Orange]

// 查找元素
int index = fruits.indexOf("Mango");  // 1

2.2 LinkedList - “手拉手的人链”

LinkedList是基于双向链表实现的List接口，同时也实现了Deque接口。

内部结构：

// 链表节点类
private static class Node<E> {
    E item;           // 元素值
    Node<E> next;     // 后继节点
    Node<E> prev;     // 前驱节点
}

// 链表大小
transient int size;

// 头节点和尾节点
transient Node<E> first;
transient Node<E> last;

工作原理：

  • 每个元素都是一个包含前后引用的节点
  • 添加元素时创建新节点并调整相关节点的引用
  • 删除元素时调整相关节点的引用

性能特点：

  • 随机访问性能较差：O(n)，需要从头/尾遍历
  • 插入/删除元素性能极佳：O(1)，只需调整引用
  • 内存占用较大，每个元素需要额外存储前后引用
  • 非线程安全

生活比喻：
LinkedList就像一群手拉手的人形成的人链：

  • 要找到第50个人，你必须从头（或尾）数过去（随机访问慢）
  • 在任何位置插入新人很容易，只需要松开两个人的手，让新人拉住他们（插入快）
  • 每个人除了自己外，还要记住前后两个人是谁（额外内存开销）
  • 人链的两端可以很方便地加入或移除人员（两端操作高效）

代码示例：

// 创建LinkedList

// 添加元素
tasks.add("Task 1");
tasks.add("Task 3");

// 在指定位置添加
tasks.add(1, "Task 2");  // [Task 1, Task 2, Task 3]

// 作为双端队列使用
tasks.addFirst("Task 0");  // [Task 0, Task 1, Task 2, Task 3]
tasks.addLast("Task 4");   // [Task 0, Task 1, Task 2, Task 3, Task 4]

// 获取元素
String first = tasks.getFirst();  // Task 0
String last = tasks.getLast();    // Task 4

// 删除元素
tasks.removeFirst();  // [Task 1, Task 2, Task 3, Task 4]
tasks.removeLast();   // [Task 1, Task 2, Task 3]

2.3 Vector - “带锁的保险柜”

Vector是List接口的古老实现类，与ArrayList类似，但是线程安全的。

内部结构：

  • 与ArrayList类似，基于数组实现
  • 方法使用synchronized关键字同步

性能特点：

  • 随机访问性能好：O(1)
  • 由于同步机制，性能比ArrayList差
  • 扩容时默认增长为原容量的2倍（而ArrayList是1.5倍）
  • 线程安全

生活比喻：
Vector就像一个带锁的保险柜，每次只允许一个人操作：

  • 查找和存取物品的方式与普通柜子相同
  • 但每次操作都需要先开锁，用完再锁上（同步操作的开销）
  • 安全性高，但效率低

适用场景：

  • 需要线程安全的动态数组
  • 多线程环境下共享集合
  • 注：现代Java应用中，通常使用Collections.synchronizedList()或CopyOnWriteArrayList代替Vector

2.4 Stack - “书堆”

Stack是Vector的子类，实现了后进先出(LIFO)的栈结构。

主要方法：

  • push(E item)：将元素压入栈顶
  • pop()：移除并返回栈顶元素
  • peek()：返回栈顶元素但不移除
  • empty()：检查栈是否为空
  • search(Object o)：查找元素在栈中的位置

生活比喻：
Stack就像一堆叠放的书：

  • 你只能从顶部放书或取书
  • 要拿到底部的书，必须先移除上面所有的书

注意：Stack类已被视为过时设计，现代Java应用通常使用Deque接口的实现（如ArrayDeque）作为栈。

    private final int pageSize;
    
    public Pagination(List<T> data, int pageSize) {
        this.data = new ArrayList<>(data);  // 创建副本，避免外部修改
        this.pageSize = pageSize;
    }
    
    public List<T> getPage(int pageNumber) {
        int startIndex = (pageNumber - 1) * pageSize;
        int endIndex = Math.min(startIndex + pageSize, data.size());
        
        if (startIndex >= data.size() || startIndex < 0) {
            return Collections.emptyList();
        }
        
        return data.subList(startIndex, endIndex);
    }
    
    public int getTotalPages() {
        return (int) Math.ceil((double) data.size() / pageSize);
    }
}

Java提供了一种结构化的方式来处理程序运行时可能出现的错误情况。

异常的基本概念

什么是异常

异常是程序运行过程中出现的非正常情况，如：

  • 程序逻辑错误
  • 系统资源不足
  • 用户操作错误
  • 设备故障等

Java使用异常对象（Exception Object）来表示这些异常情况，并提供了处理这些异常的机制。

异常的分类

Java中的异常主要分为三大类：

  1. 检查型异常（Checked Exception）

    • 编译时异常，必须显式处理
    • 继承自Exception类但不是RuntimeException的子类
    • 例如：IOException, SQLException

  2. 非检查型异常（Unchecked Exception）

    • 运行时异常，不强制处理
    • 继承自RuntimeException
    • 例如：NullPointerException, ArrayIndexOutOfBoundsException

  3. 错误（Error）

    • 表示严重问题，通常无法恢复
    • 继承自Error类
    • 例如：OutOfMemoryError, StackOverflowError

Java异常类层次结构

Throwable (所有异常和错误的父类)
├── Error (严重错误，通常无法恢复)
│   ├── OutOfMemoryError
│   ├── StackOverflowError
│   └── ...

    │   └── ...

异常处理语法

try-catch-finally

基本语法：

try {
    // 可能抛出异常的代码
} catch (ExceptionType1 e1) {
    // 处理ExceptionType1类型的异常
} catch (ExceptionType2 e2) {
    // 处理ExceptionType2类型的异常
} finally {
    // 无论是否发生异常，都会执行的代码
    // 通常用于释放资源
}

示例：

public void readFile(String filename) {
    FileInputStream fis = null;
    try {
        fis = new FileInputStream(filename);
        int data;
        while ((data = fis.read()) != -1) {
            System.out.print((char) data);
        }
    } catch (FileNotFoundException e) {
        System.err.println("文件不存在: " + e.getMessage());
    } catch (IOException e) {
        System.err.println("读取文件错误: " + e.getMessage());
    } finally {
        // 关闭资源
        if (fis != null) {
            try {
                fis.close();
            } catch (IOException e) {
                System.err.println("关闭文件错误: " + e.getMessage());
            }
        }
    }
}

let num2 = 5;

console.log(++num1 * 2); // 12 (先自增到6，再乘2)
console.log(num2++ * 2); // 10 (先乘2，再自增到6)
console.log(d++ + ++d +d) //34
// 让我们一步步分解这个表达式：
// 1. d++ + ++d + d
//    - d++ 是后置自增：先使用d的值(10)，然后d变成11
//    - ++d 是前置自增：先将d加1变成12，再使用12
//    - 最后的d 直接使用当前值12
// 所以计算过程是：10 + 12 + 12 = 34

let d = 10;
d = d++;
console.log(d);    // 结果：10
// 让我们分析这个过程：
// 1. d = d++
//    - d++ 是后置自增，会先返回原始值10
//    - 然后d会增加到11
//    - 但是这个11还没有被使用，d就被赋值了
//    - 赋值运算符=会将d++返回的原始值10赋给d
//    - 所以最终d的值仍然是10

区别总结：

  1. ++a（前置自增）：

    • 先将a加1
    • 再返回加1后的值
    • 常用于：需要立即使用增加后的值时

  2. a++（后置自增）：

    • 先返回原始值
    • 再将a加1
    • 常用于：需要在使用当前值后再增加时

最佳实践：

  1. 单独使用时，++a 和 a++ 效果相同
  2. 在表达式中使用时要特别注意前置和后置的区别

  1. null和undefined：

    • null == undefined 返回 true
    • null和undefined与其他值比较都返回false

  2. 布尔值参与比较：

    • 然后再和另一个操作数比较

  3. 数字和字符串比较：

    • 字符串转换为数字
    • 如果转换失败则变成NaN

  4. 对象和原始类型比较：

    • 对象先调用valueOf()或toString()转换为原始类型
    • 然后按照上述规则进行比较

// 基本类型：存储在栈内存中

// 引用类型：比较引用地址

1. 引用类型的拷贝

// 方法一：Object.assign()
let original = { name: "张三", info: { age: 20 } };

// 方法二：展开运算符
let shallowCopy2 = { ...original };

// 2. 深拷贝
// 方法一：JSON转换（有局限性）
let deepCopy1 = JSON.parse(JSON.stringify(original));

// 方法二：递归实现
function deepClone(obj) {
    if (obj === null || typeof obj !== 'object') return obj;
    let copy = Array.isArray(obj) ? [] : {};
    for (let key in obj) {
        if (obj.hasOwnProperty(key)) {
            copy[key] = deepClone(obj[key]);
        }
    }
    return copy;
}

六、apply 和 call 方法

用途

apply 和 call 都是函数对象的方法，用于改变函数执行时的上下文（即改变函数内部的 this 指向）。

语法

func.call(thisArg, param1, param2, ...)
func.apply(thisArg, [param1, param2, ...])

区别

  • call 方法：参数需要一个一个传递
  • apply 方法：参数以数组形式传递

示例

1. 基本使用

const person = {
    name: '张三',
    sayHi: function(greeting) {
        console.log(`${greeting}，我是${this.name}`);
    }
};

const other = {
    name: '李四'
};

// 使用 call
person.sayHi.call(other, '你好');  // 输出：你好，我是李四

// 使用 apply
person.sayHi.apply(other, ['你好']); // 输出：你好，我是李四

2. 实际应用场景

2.1 借用数组方法
// 将类数组转换为真正的数组
function convert() {
    return Array.prototype.slice.call(arguments);
}
console.log(convert(1, 2, 3)); // 输出：[1, 2, 3]

// 或使用 apply 找出数组最大值
const numbers = [5, 6, 2, 3, 7];
const max = Math.max.apply(null, numbers);
console.log(max); // 输出：7
2.2 继承
function Animal(name) {
    this.name = name;
}

function Cat(name, color) {
    // 调用 Animal 构造函数，并将 this 指向 Cat 实例
    Animal.call(this, name);
    this.color = color;
}

const cat = new Cat('咪咪', '白色');
console.log(cat.name, cat.color); // 输出：咪咪 白色

关键点

  1. this 绑定

    • 第一个参数决定了函数内部的 this 指向
    • 如果第一个参数为 null 或 undefined，this 会指向全局对象（非严格模式）或保持为 null/undefined（严格模式）

  2. 参数传递

    • call：func.call(thisArg, arg1, arg2, ...)
    • apply：func.call(thisArg, [arg1, arg2, ...])

  3. 使用场景

    • 借用其他对象的方法
    • 继承时调用父类构造函数
    • 调用需要特定 this 值的函数

最佳实践

  1. 当参数数量固定时，优先使用 call，代码更直观
  2. 当参数数量不固定或参数本身就是数组时，使用 apply
  3. 在现代 JavaScript 中，可以考虑使用展开运算符 ... 代替 apply：
    const numbers = [5, 6, 2, 3, 7];
const max = Math.max(...numbers); // 替代 Math.max.apply(null, numbers)

注意事项

  1. 在严格模式下，如果第一个参数为 null 或 undefined，this 不会被转换为全局对象
  2. 这两个方法都会立即执行函数
  3. 如果需要稍后执行，可以考虑使用 bind 方法

JavaScript垃圾回收机制详解📚

一、基本概念

1. 什么是垃圾回收

JavaScript会自动进行内存管理和垃圾回收（Garbage Collection，简称GC），主要目的是：

  • 识别不再使用的内存（垃圾）
  • 回收这些内存供后续使用
  • 防止内存泄漏

// 标记-清除算法的工作示例

};

工作步骤：

  1. 垃圾回收器找到所有根对象（全局对象）
  2. 标记从根对象可达的所有对象
  3. 清除未被标记的对象

2. 引用计数算法（Reference Counting）

// 引用计数示例

    name: "测试"

reference = null;    // 引用计数 = 0，可以被回收

用途

    // 遍历该元素后的所有元素依次比较

            j--; 

        }
    }
}
