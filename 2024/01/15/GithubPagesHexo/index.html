    一、安装环境

    Nodejs安装

    • Nodejs下载入口

    • 修改全局模块下载路径。

      // 命令指定到路径
npm config set prefix "D:\workspace\node_global"
npm config set cache "D:\workspace\node_cache"

    • 用户变量Path修改为

      D:\workspace\node_cache

    • 系统变量新建：

      变量名：NODE_PATH
变量值：D:\workspace\node_global\node_modules

    • 系统变量Path修改为

      D:\workspace\node_global

    • 扩权：

    • 验证下载到自定义目录

      npm install express -g     # -g是全局安装的意思

    • 更换npm源为淘宝镜像

      npm默认的registry即下载npm包是从国外服务器下载，遂更换为淘宝源。

      npm config set registry https://registry.npm.taobao.org/ # 淘宝源
npm install -g cnpm --registry=https://registry.npm.taobao.org # 全局安装基于淘宝源的cnpm

    Git安装

    • Git下载入口

      第一种是仅从 Git Bash 使用 Git。这个的意思就是你只能通过 Git 安装后的 Git Bash 来使用 Git ，其他的什么命令提示符啊等第三方软件都不行。

      第二种是从命令行以及第三方软件进行 Git。这个就是在第一种基础上进行第三方支持，你将能够从 Git Bash，命令提示符(cmd) 和 Windows PowerShell 以及可以从 Windows 系统环境变量中寻找 Git 的任何第三方软件中使用 Git。推荐使用这个。

      第三种是从命令提示符使用 Git 和可选的 Unix 工具。选择这种将覆盖 Windows 工具，如 “ find 和 sort ”。只有在了解其含义后才使用此选项。一句话，适合比较懂的人折腾。

    Hexo安装

    • 博客根目录【D:\workspace\hexo_blog】

      # 安装hexo
$ npm install hexo-cli -g 
$ npm install hexo-deployer-git --save

    • 根目录下新建hexo目录【D:\workspace\hexo_blog\hexo】

      # 初始化
hexo init
hexo generate
hexo server

    关于本地git通过ssh链接github时 time out问题的解决方法

    Host github.com
HostName ssh.github.com  # 这是最重要的部分
User git
Port 443
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa
